¿Qué es jQuery?

    jQuery puede considerarse que está compuesto por unas librerías que contienen una serie de funciones JavaScript listas para usar. Entre otras cosas, implementa una serie de clases que permiten programar sin tener en cuenta el navegador, puesto que funcionan igual para las plataformas más habituales.

    Permite la creación de efectos dinámicos, galerias, aplicaciones con Ajax, etc. de forma más simple. Sencillamente habrá que conocer estas librerías y programar haciendo uso de las clases, propiedades y métodos adecuados.

    Estas librerías (framework) son gratuitas y podrán utilizarse incluyendo en las páginas un script JavaScript con el código jQuery, descargable desde su web. El archivo ocupa unos 56 KB y si nuestro servidor lo envía comprimido el peso de jQuery se queda en unos 19 KB. El fichero será enviado por el servidor al cliente una vez por usuario, después queda en la caché.

    Goza de gran aceptación por parte de los programadores, es estable y está bien documentado. Requiere habilidades avanzadas de programación y conocimientos de programación orientada a objetos.

Ejemplos de jQuery

    Veamos el poco código que hay que escribir para conseguir algunos dinamismos simples.

    En éste primer ejemplo existen dos botones y un texto que cambia al pulsar cualquiera de los botones. El resultado puede verse pulsando aquí. El código del ejemplo es el siguiente:

<div id="capa" style="padding: 10px; background-color: #ff8800">
    Haz clic en un botón
</div>
<input type="button" value="Botón A"
      onclick="$('#capa').html('Has hecho clic en el botón <b>A</b>')">
<input type="button" value="Botón B"
      onclick="$('#capa').html('Recibido un clic en el botón <b>B</b>')">

    El código consta de una capa y dos botones que responden al evento onclick y ejecutan una instrucción JavaScript que hace uso de herramientas jQuery que selecciona la capa (DIV) y ejecuta sobre ella un método (html) que cambia el contenido HTML del div.

    En el segundo ejemplo vamos a disponer de dos capas, una siempre visible y la otra que se mostrará u ocultará según tengamos el puntero del ratón sobre la primera o fuera de ella. El resultado puede verse pulsando aquí y el código es el siguiente:

<div id="capa" style="padding: 10px; background-color: #ff8800;">
    Pon el ratón encima de esta capa
</div>
<br>
<div id="mensaje" style="display: none;">
    Has puesto el ratón encima!! <br>(Ahora quítalo de la capa)
</div>
$("#capa").mouseenter(function(evento){
      $("#mensaje").css("display", "block");
});
$("#capa").mouseleave(function(evento){
      $("#mensaje").css("display", "none");
});

    Así se han gestionado los dos eventos del DIV con id="capa". Además, se ha definido el código de los eventos por medio de funciones, que se encargarán de mostrar u ocultar la segunda capa con id="mensaje". En cada función se selecciona la capa con id "mensaje" y con el método css() se cambia a la propiedad "display" entre los valores "block" y "none".

Instalación

    Para descargar la última versión del framework puede accederse a la página de jQuery. En el momento de escribir este resumen la release actual es la 1.9.1. Hay dos posibilidades para descargar, una que le llaman PRODUCTION, que es la adecuada para páginas web en producción, puesto que está minimizada y ocupa menos espacio, con lo que la carga de nuestro sitio será más rápida. La otra posibilidad es descargar la versión que llaman DEVELOPMENT, que está con el código sin comprimir, con lo que ocupa más espacio, pero se podrá leer la implementación de las funciones del framework, que puede ser interesante en etapa de desarrollo, porque podremos bucear en el código de jQuery por si tenemos que entender algún asunto del trabajo con el framework.

    La descarga consiste en un archivo con extensión js que contiene el código completo de jQuery y que deberá incluirse en nuestras páginas web de la misma forma que podemos insertar un fichero de estilos css:

<script src="jquery-1.9.1.min.js" type="text/javascript"></script>

    El nombre del fichero js debe coincidir con el descargado. Así quedan incluidas todas las funciones de jQuery en la página.

Comprobar que la página está cargada

    Para hacer uso de DOM se hace necesario detectar el momento en que la página está completamente cargada. Si no se espera a que la página esté lista para recibir comandos JavaScript que utilicen la estructura del documento se corre un alto riesgo de que se produzcan errores de JavaScript en tiempo de ejecución.

    Generalmente, cuando se desea ejecutar Javascript después de la carga de la página, si no utilizamos ningún framework, lo más normal será utilizar un código como este:

window.onload = function () {
    alert("cargado...");
}

    Pero esta sentencia, que carga una funcionalidad en el evento onload del objeto window, sólo se ejecutará cuando el navegador haya descargado completamente TODOS los elementos de la página, lo que incluye imágenes, iframes, banners, etc. lo que puede tardar bastante, dependiendo de los elementos que tenga esa página y su peso. En realidad no hace falta esperar todo ese tiempo de carga de los elementos de la página para poder ejecutar sentencias Javascript que alteren el DOM de la página. Sólo habría que hacerlo cuando el navegador ha recibido el código HTML completo y lo ha procesado al renderizar la página. Para ello, jQuery incluye una manera de hacer acciones justo cuando ya está lista la página, aunque haya elementos que no hayan sido cargados del todo. Esto se hace con la siguiente sentencia:

$(document).ready(function(){
    //código a ejecutar cuando el DOM está listo para recibir instrucciones.
});

    Con $(document) se obtiene una referencia al documento (la página web) que se está cargando y con el método ready() se define un evento, que se produce al quedar listo el documento para realizar acciones sobre el DOM de la página.

Gestión de un enlace "a href" desde jQuery

    El funcionamiento de "a href" es sobradamente conocido. Para crear un evento click desde jQuery que muestre un mensaje hay que pasar el código correspondiente como función:

$("a").click(function(evento){
   //aquí el código que se debe ejecutar al hacer clic
});

    Con $("a") se hace referencia a las etiquetas "A" (enlaces) del documento; como sólo hay un enlace se hace referencia a él.

    El método click() sobre $("a") define el evento y dentro de la función debe especificarse el código que debe ejecutarse cuando se produzca el mismo. El código completo sería:

$(document).ready(function(){
   $("a").click(function(evento){
      alert("Has pulsado el enlace.\nAhora serás enviado a DesarrolloWeb.com");
   });
});

    Con este código se muestra el mensaje "Has pulsado ..." al hacer click sobre el enlace. El comportamiento puede comprobarse pulsando aquí.

    El comportamiento por defecto de un enlace consiste en que al pulsar sobre el mismo nos lleva auna URL. Con el código anterior, primero se ejecuta el código colocado para el evento click y posteriormente el navegador nos lleva a la URL. El comportamiento por defecto se puede bloquear añadiendo una llamada al método preventDefault() sobre el evento.

    En el código anterior, la función definida para el evento click sobre el enlace recibe un parámetro que es un manejador de evento, y que tiene sus propios métodos y propiedades. preventDefault es uno de esos métodos y un ejemplo de uso es el siguiente:

$(document).ready(function(){
   $("a").click(function(evento){
      alert("Has pulsado el enlace pero ...\nsu funcionamiento está anulado");
      evento.preventDefault();
   });
});

    El resultado puede comprobarse pulsando aquí.
