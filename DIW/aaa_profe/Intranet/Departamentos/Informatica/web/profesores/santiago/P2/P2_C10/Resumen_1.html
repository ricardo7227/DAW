El diseño mediante la propiedad display

    Existen 4 modelos de posicionamiento de los elementos dentro de la página web: estático, relativo, absoluto y fijo. Su comportamiento puede diferir de unos navegadores a otros.

Propiedades relacionadas con el posicionamiento

    Las principales propiedades de esta categoría son las siguientes:
  • position.- esta propiedad no define la posición sino el modo en que se colocan los elementos de un documento HTML dentro de la ventana del navegador. Los valores permitidos son los siguientes:

    • static.- valor predeterminado. El elemento se coloca según el flujo natural de los elementos, sin afectarle el valor de otras propiedades como top, bottom, etc.
    • fixed.- el elemento se colocará en una posición fijada por las propiedades de distancia a los bordes de la ventana (top, bottom, left y right). Los elementos siguientes se visualizarán como si éste no existiera, aunque se situará sobre los demás, como si fuera el último en colocarse. Además, no se desplaza con las barras de desplazamiento.
    • relative.- el elemento se colocará en una posición relativa a la que le correspondería en el posicionamiento por defecto (estatico) y en base a distancias indicadas por las propiedades top, bottom, left o right, que pueden tener valores positivos o negativos. Los elementos siguientes dejarán el espacio correspondiente al elemento como si su posición fuera estática.
    • absolute.- el elemento se colocará en una posición relativa al primer elemento padre (de nivel superior en la jerarquía) que tenga un posicionamiento diferente a static y en base a distancias indicadas por la propiedad top, bottom, left o right, que pueden tener valores positivos o negativos. Los demás elementos se comportan de forma similar a fixed.

  • top, bottom, left y right.- establecen el margen del elemento respecto a su posición normal. Pueden tener los valores auto (predeterminado y que asume el valor normal que le corresponde al elemento), unidades de longitud o porcentajes referidos al elemento contenedor. Por ejemplo: top: 0; left: 0; position: absolute; corresponde a la esquina superior izquierda de la ventana.

  • z-index.- indica el nivel de profundidad en la colocación superpuesta de elementos en la ventana (pila de elementos). Cuanto mayor sea el nivel asignado más arriba aparecerá.

  • visibility.- especifica si el elemento es visible o no. El valor predeterminado es visible. Los demás valores posibles son: hidden (oculto) o collapse (aplicable para ocultar filas y columnas de tablas).

Posicionamiento de los elementos

    El posicionamiento de los elementos se controla con cuatro propiedades específicas que indican las distancias a cada uno de los cuatro lados: top, bottom, left y right, cuyo comportamiento depende fundamentalmente del valor de la propiedad position.

    Para informar cualquiera de estas cuatro propiedades se utiliza el formato: lado: distancia/porcentaje. Por ejemplo:

          top: 50px;
          left: 30px;

    El posicionamiento estático (static) es el más sencillo porque el elemento se coloca en el sitio que le corresponde según su flujo natural. Las propiedades top, bottom, etc. no se tienen en cuenta.

    El posicionamiento relativo (relative) se utiliza para colocar el elemento en una posición diferente a la que le tocaría siguiendo el flujo natural. La nueva posición utiliza las propiedades top, left, etc. para desplazar el elemento respecto al lugar que ocuparía normalmente. Debe observarse que se respeta el espacio que el elemento ocuparía si su posicionamiento fuera estático. Si se utilizan porcentajes, serán relativos a las longitudes del propio elemento, es decir, con "right: 10%" el elemento se mostrará un diez por ciento de su propia longitud desplazado hacia la izquierda de su posición normal.

    El posicionamiento absoluto es similar al relativo con la diferencia que las distancias son relativas al borde de la ventana de visualización. No se respeta el espacio que el elemento ocuparía si tuviera un posicionamiento estático. Si los valores se dan en porcentajes serán relativos a la longitud del propio elemento (?).

    Finalmente, el posicionamiento fijo es similar al absoluto (en cuanto a posicionamiento) con la diferencia de que éste no varía su posición cuando se mueven las barras de desplazamiento. No se respeta el espacio que el elemento ocuparía si tuviera un posicionamiento estático.

    Las propiedades de posicionamiento determinan el tamaño de un elemento en el caso de que no se especifique el tamaño del mismo pero sí, por ejemplo, las propiedades left y right (o top y bottom (?)).

Flotación de los elementos

    La propiedad float permite definir el posicionamiento de un elemento de un modo que modifica el flujo normal del contenido de la página. Cuando se utiliza la propiedad float el elemento flota hasta el margen especificado. Sus posibles valores son: left, right y none. El valor none equivale al flujo natural de posicionamiento de los elementos en la página.

    Lo interesante de esta propiedad es que si hay otros elementos en la página tienden a colocarse en el margen opuesto, quedando la página con menos espacios en blanco.

    Si se desea evitar la flotación de un elemento de forma que su posición no dependa de la flotación del elemento anterior, puede utilizarse la propiedad clear, vista anteriormente (para colocar texto alrededor de imágenes) y que admite los valores: right, left y both.

    La propiedad float se puede utilizar para crear columnas de elementos, basta con utilizar en cada uno de ellos "float: left".

Control del tamaño de un elemento

    El tamaño de un elemento se puede controlar desde distintas propiedades, dependiendo en parte del tipo de elemento (por ejemplo, no es lo mismo una imagen que un texto). No obstante, en CSS existe un grupo de propiedades específicas de uso general para determinar el tamaño de un elemento, como son width y height.

    CSS también permite definir límites mínimos y máximos para el tamaño e los elementos, que pueden impedir la visualización incorrecta de algunos elementos. Las propiedades disponibles son:
  • min-width.- ancho mínimo.
  • max-width.- ancho máximo.
  • min-height.-altura mínima.
  • max-height.- altura máxima.
    Los valores que se pueden asignar a estas propiedades son similares a los de width y height.

    Si el tamaño de un elemento se define de modo independiente al contenido, puede existir el riesgo de que el área contenedora quede pequeña y se produzca una situación de overflow. La propiedad CSS que permite controlar este comportamiento es overflow, que puede tomar los valores:
  • visible.- el contenido se muestra completo aunque exceda su área contenedora.
  • hidden.- el contenido se recorta para limitarlo a su área contenedora. La parte que quede fuera de la parte recortada no puede visualizarse de ninguna manera.
  • scroll.- el contenido se muestra dentro de su área contenedora y se habilitan barras de desplazamiento para poder visualizar todo el contenido.
  • auto.- el comportamiento se deja en manos del agente de usuario (en base a su configuración de la condición de overflow).

Superposición de elementos

    CSS permite la superposición de elementos, formando pilas de elementos que ocupan áreas comunes de visualización. Para ello se utiliza un tercer eje (el eje z) que permite percibir el documento como una pila de hojas.

    Para controlar la pila de elementos se utiliza la propiedad z-index, cuyo formato es:

          z-index: valor;

    El valor es un número que indica la posición en la pila: 0 para el de más abajo, 1 para el siguiente, 2 para el siguiente, etc. También se pueden asignar valores negativos. Si no se especifica se presupone el valor 0.

Control de la visibilidad del elemento

    La propiedad visibility permite controlar si un elemento es visible o no. Admite los valores:
  • visible.- valor por defecto.
  • hidden.- el elemento no se muestra.
  • collapse.- en elementos de filas o columnas contrae los bordes. Si el elemento no tiene filas ni columnas su comportamiento es el mismo que hidden.
    Otra manera de ocultar un elemento es mediante el uso de la propiedad display con el valor none, como se muestra en el apartado siguiente. La diferencia que existe es que con "visibility: hidden;" el elemento sigue ocupando espacio en la página mientras que con "display: none;" no se genera su caja contenedora y, por lo tanto, no ocupa ningún espacio.

    "display: none;" se complementa generalmente con código JavaScript, que permite mostrar/ocultar elementos cuando el usuario realiza alguna acción, por ejemplo, al pulsar un botón.

El diseño mediante la propiedad display

    La propiedad display define el tipo de caja que generará el elemento. Admite los siguientes valores:
  • inline.- valor por defecto. Genera una caja en la misma línea, sin saltos de línea ni antes ni después del elemento. Un elemento inline ocupa solo el espacio necesario que necesita el contenido.
  • none.- no se genera ningún tipo de caja, por lo que el elemento no se visualiza.
  • block.- se genera una caja de bloque con un salto de línea antes y otro después del elemento.Un elemento de bloque ocupa todo el espacio hasta el final de la línea, aunque su contenido no ocupe todo el espacio.
  • list-item.- el elemento generará una caja de bloque (block) y una de línea (inline) para la marca de lista.
  • table.- el elemento se comporta como una tabla, con un salto de línea antes y después del elemento.
  • inherit.- el elemento hereda el valor de la propiedad del elemento padre.
  • inline-block.- se genera una caja de bloque presentada como una caja inline. Lo interesante de esta opción es que al ser un bloque se puede informar medidas de tamaño.
  • inline-table.- se genera una caja en línea sin saltos de línea, como la etiqueta <table>.
  • run-in.- se genera una caja en línea o una caja de bloque dependiendo del contexto.
  • table-caption.- el elemento se comporta como un encabezamiento de tabla (<caption>).
  • table-cell.- el elemento se comporta como una celda de una tabla.
  • table-column.- el elemento se comporta como una columna de una tabla.
  • table-column-group.- el elemento se comporta como un grupo de columnas de una tabla (<colgroup>).
  • table-footer-group.- el elemento se comporta como un pie de grupo de filas de una tabla.
  • table-header-group.- el elemento se comporta como un encabezado de grupo de filas de una tabla.
  • table-row.- el elemento se comporta como una fila de tabla.
  • table-row-group.- el elemento se comporta como un grupo de filas de una tabla.
    Como se ve, la propiedad display tiene un conjunto de valores relacionados con la creación de tablas que permite presentar contenido en forma de tabla prescindiendo de las etiquetas table, tr, etc. de HTML. Ver ejemplos de la página 336 y sucesivas.
