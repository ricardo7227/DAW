Este resumen se ha realizado sobre los manuales de desarrolloweb.com.

    Para trabajar con jQuery puede incluirse el script de la librería en el HEAD de la página o utilizar el servicio CDN de jQuery, o de otros proveedores como Google Code, lo que comporta ciertas ventajas y ciertos inconvenientes. CDN significa Content Delivey Network (Red de entrega de contenido) y es un servicio que permite incluir las l ibrerías de código jQuery desde los servidores de algunas importantes empresas. Cuando se usa un CDN, en vez de enlazar con los scripts de jquery alojados en nuestro sitio web, enlazamos con una URL de otro dominio que los tiene alojados. Esto permite disponer de mayor velocidad de entrega de nuestro sitio web.

Servicios CDN de jQuery

    Diversas empresas ofrecen el código de jQuery para enlazarlo en nuestros sitios web basados en el framework. Los podemos enlazar directamente de esos servidores, haciendo lo que se denomina "hotlinking". En este caso, sería un hotlinking deseable, puesto que estos servidores CDN están pensados justamente para ello. Esta técnica de "linkado en caliente" no debe utilizarse con recursos como imágenes porque puede considerarse como un robo de ancho de banda.

    Google Code: Contiene links a los frameworks Javascript más populares, con CDN de jQuery, Mootools, Prototype y muchos otros. Ver http://code.google.com/intl/es-ES/apis/libraries/. En el momento de escribir este resumen, la versión que ofrecen como hosting CDN es la 1.11.0 y se enlaza en la URL http://ajax.googleapis.com/ajax/libs/jquery/1.11.0/jquery.min.js.

    Microsoft CDN: Dispone de una red de contenido distribuido con CDN de varias librerías Javascript y Ajax, como jQuery o Modernizr. Encontramos más información en la dirección http://www.asp.net/ajaxlibrary/cdn.ashx . La última versión que está disponible en estos momentos es http://ajax.aspnetcdn.com/ajax/jquery/jquery-1.9.0.min.js

 CDN jQuery: el propio jQuery ofrece una red CDN para descarga de sus librerías, a partir de la ruta http://code.jquery.com/jquery-1.7.1.min.js

    Los CDN tienen habitualmente soporte para transferencia de las librerías por HTTP y HTTPS, así como para la descarga de las librerías en versión minimizada (indicada para sitios en producción) o estándar (indicada para sitios en desarrollo). Debes consultar los servicios CDN para obtener más información.

Ventajas e inconvenientes de los CDN

    Algunas ventajas del uso de CDN's:
  • Mayor velocidad de entrega: Los servicios CDN están ofrecidos por grandes empresas, con replicación de servidores y diversas localizaciones de entrega a lo largo del mundo. Posiblemente Google, o cualquiera de los otros provedores CDN, pueda enviar el script jQuery más rápido que tu propio servidor y lo distribuya desde una localización más cercana a la red del cliente que te visita.
  • Cacheado probable: Es muy probable que la persona que te visita ya haya cacheado el script jQuery, tras la visita a otra página web que esté usando también el CDN de alguna de estas empresas. Por ello quizás no tenga ni que esperar a descargar el framework y utilice la copia que ya tiene en la caché del navegador.
    Algunos inconvenientes:
  • Necesitamos estar conectados a Internet para acceder al CDN: Durante el desarrollo del sitio web, si estamos offline, sería imprescindible acceder a la copia en local de las librerías para que nuestra web funcione. Si necesitamos probar la web en entornos donde quizás no dispongamos de Internet (por ejemplo, la oficina de un cliente donde no sabemos si van a estar "online"), nos interesaría acceder a las librerías por medio de la copia en local.
  • Tenemos menor control: No se puede tener control total sobre lo que se descarga como script. Claro que el script estará correcto, pero no podrás modificarlo si lo necesitas, ya que está en otro servidor. Esto generalmente no será un problema, pero hay algunos marcos en los que en la práctica sí interesa tener un poco más de control. Por ejemplo, en el caso de algunas librerías que permiten incluirse solo parcialmente (por ejemplo en Mootools somos capaces de seleccionar qué módulos del framework deseamos, o en jQueryUI qué widgets queremos utilizar), no será posiblea a través de la versión que nos traemos por el CDN.

Optimizar el uso de CDN

    Por lo general las ventajas del uso de CDN superan a los inconvenientes. Puede utilizarse algún script sencillo que permita utilizar el CDN si está activo y una copia local en caso contrario. Pueden verse varios scripts, cada uno con sus ventajas e inconvenientes pinchando aquí.

    Uno de los scripts más sencillos es el siguiente:

<script src="//ajax.googleapis.com/ajax/libs/jquery/1.7.1/jquery.min.js"></script>
<script>
  window.jQuery || document.write('<script src="js/jquery-1.7.1.min.js">x3C/script>')
</script>
