Impresión con CSS

Introducción

La Web es un medio para la presentación de todo tipo de documentos y CSS contribuye para que cada tipo de documento se pueda formatear del modo más adecuado para su visualización en los distintos tipos de pantallas gráficas. Este proceso es el que hemos estudiado hasta el momento en los capítulos anteriores pero ahora debemos observar las necesidades de los usuarios que quieren que el contenido en pantalla se vuelque hacia una salida impresa; esto nos ocurre frecuentemente al imprimir recibos, facturas y documentos en general que queremos mantener en papel.

La impresión en papel es una de las principales opciones de salida y sobre la que ya hay décadas de experiencia pero no es la única posibilidad; un documento web se puede mostrar en distintos dispositivos de presentación, por ejemplo, teléfonos móviles, y esto representa un nuevo desafío para CSS y para los diseñadores: conseguir que un documento web mantenga el máximo nivel de calidad de presentación en los distintos medios.

CSS dispone de un grupo de funcionalidades que permite personalizar el formateo de un documento para los distintos tipos de medios. Por lo tanto, el lector debe considerar que la salida impresa es sólo una de las muchas posibilidades de formateo del documento; por lo que en este capítulo lo que realmente estudiaremos no es sólo la salida impresa sino algo más general: cómo gestiona CSS la presentación en los distintos tipos de medios.

Tipos de medios

CSS nos permite definir estilos orientados a un determinado tipo de medio o un grupo de tipos de medios. Cuando no se especifica nada se presupone que el estilo es válido para todos los medios (all); por ejemplo, si al definir un estilo lo hacemos para que sólo se utilice en pantallas de ordenador, se indicará que el tipo de medio es screen.

Propiedad margin

Los márgenes componen la parte externa de la página y esto no debe confundirse con el margen que se puede definir a nivel de la etiqueta <BODY>. Para asignar el margen de la página se utiliza la propiedad margin del mismo modo que se codifica en los ejemplos vistos anteriormente:

@page {size: 17cm 21cm; margin: .8cm;}

No todos los navegadores soportan la regla @page por lo que debemos comprobar su funcionamiento antes de depender de que nuestro código dependa de la regla.

Marcas de corte

Dentro del entorno de la publicación se utilizan lo que se denominan marcas de corte, que señalan la posición de corte. Estas marcas son signos + que se imprimen en los bordes de la página. Las marcas de cruces son otros símbolos que se imprimen cerca del borde de las páginas y sirven para alinear las páginas.

La propiedad marks, que aún no está soportada por los principales agentes de usuario, tiene la siguiente sintaxis:

marks: <crop> | <cross> | <none> | <inherit>;

Control de los saltos de página

La impresión de documentos siempre debe saber gestionar los saltos de página, sea porque el contenido no cabe en la página física o porque se quiere provocar un cambio de página. CSS incluye tres propiedades para controlar los saltos de página:

  • page-break-before: Determina cómo se produce el salto de página antes de un determinado elemento y en qué lado continua el flujo de la página siguiente.
  • page-break-after: Determina cómo se produce el salto de página después de un determinado elemento y en qué lado continua el flujo de la página siguiente.
  • page-break-inside: Determina cómo se produce el salto de página desde dentro de un elemento. Por ahora, esta propiedad está soportada sólo por Opera.
