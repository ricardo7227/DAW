Definición de base de datos

    Una base de datos es una colección de datos con información de cualquier tipo.

Motivos para usar una base de datos

    Algunos conceptos que justifican el uso de bases de datos pueden ser:
  • Almacenamiento.- puede almacenarse electrónicamente.
  • Accesibilidad.- depende del formato en que se guarde.
  • Organización.- pueden utilizarse claves o índices que eviten duplicidad de datos y faciliten las búsquedas.
  • Manipulación.- facilita la modificación, borrado e inserción de datos.
    Resumiendo, las bases de datos facilitan la manipulación, localización y organización de los datos.

Conceptos básicos sobre bases de datos

    Las bases de datos pueden consistir en uno o varios archivos que contienen los datos y otros archivos con los índices. Los archivos de datos se pueden estructurar en otros archivos llamados tablas que a su vez están compuestos por campos.

    Las tablas de las bases de datos almacenan datos sobre un tipo de objeto o entidad (personas, contratos, movimientos, etc.). Las tablas a su vez están formadas por registros y estos por campos.

SGBD: Sistemas de gestión de bases de datos.

    Un sistema de gestión de bases de datos representa una infraestructura con la que se almacenan, recuperan y administran los datos de una base de datos. Además, debe mantener la integridad de los datos almacenados, es decir, controlar que los datos se guardan manteniendo unas condiciones establecidas de vinculación y cumpliendo las restricciones que se hayan definido. También se ocupan de suministrar la conectividad con una amplia gama de lenguajes de programación y de garantizar la seguridad del acceso a los datos implementando el control de acceso en función de los permisos de cada usuario.

    Bases de datos relacionales

    Las tablas que componen una base de datos deben organizarse de manera que no pierdan las relaciones existentes entre los datos. Como guia de diseño de una base de datos deben seguirse las siguientes pautas:
  • En cada fila de datos no deben aparecer grupos repetitivos.
  • Todos los elementos de una determinada columna son del mismo tipo de dato y representan la misma clase de dato.
  • Cada columna tiene su nombre, que es el nombre del campo.
  • Todas las filas de una tabla son diferentes.
  • Cada fila debe tener un campo o conjunto de campos que sirva de clave de identificación.
    Tipos de relaciones

    Entre las tablas de una base de datos pueden existir las siguientes relaciones:
  • Uno a uno.- un registro de una tabla solo se vincula con un registro de otra.
  • Uno a muchos.- un registro de una tabla se vincula con varios de otra.
  • Muchos a muchos.- varios registros de una tabla se vinculan con varios de otra.
    Claves

    Las claves se utilizan como identificadores de los registros y para clasificar las consultas. Existen dos tipos de claves: claves primarias y claves externas. Las claves primarias son únicas y pueden estar compuestas por varios campos. Las claves externas se utilizan en las relaciones como valor de búsqueda en las tablas hijas.

    Índices

    Los índices se utilizan para especificar campos de búsqueda que no son claves primarias ni externas. La creación de índices incrementa la carga de procesamiento y de accesos al disco en el momento de la actualización de la tabla, pero si interviene en muchas consultas puede resultar ventajoso su uso.

    Es conveniente que los índices correspondan a columnas con no demasiados caracteres.

    En cualquier caso, a la hora de diseñar tablas deben tenerse en consideración los siguientes tipos de datos: índices, claves primarias, claves externas, campos que se usan en combinación de tablas (joins), campos sobre los que se realizan búsquedas (where) y campos sobre los que se realizan clasificaciones (order by).

    Integridad de datos

    La integridad de datos significa que en la base de datos no hay datos obsoletos o incorrectamente relacionados debido a actuaciones incompletas o fallidas. La integridad de datos se puede asegurar mediante la comprobación de uno o más de los siguientes mecanismos:
  • Integridad de entidades.- ningún valor de la clave primaria puede ser nulo.
  • Integridad referencial.- ningún registro de una tabla hija tendrá una clave externa que no exista como clave primaria en la tabla padre.
  • Integridad de dominio.- los valores de un campo cumplen con una descripción de dominio (longitud, formato, ...).
    Guia para la normalización de bases de datos

    El proceso de optimización del diseño de una base de datos se denomina normalización de la base de datos. La normalización evita las inconsistencias, los errores de actualización y ayuda a eliminar las redundancias. Se considera que una base de datos tiene un diseño normalizado cuando se alcanza al menos el tercer nivel de normalización o tercera forma normal:
  • Primera forma normal.- cada campo solo contiene un valor. No hay campos que contengan tablas.
  • Segunda forma normal.- todos los atributos no claves dependen de la clave principal. En caso contrario debería generarse una nueva tabla con los campos que no dependen de la clave principal.
  • Tercera forma normal.- todos los atributos no claves dependen de manera no transitiva de la clave principal. En caso contrario deberá crearse una nueva tabla con los campos que dependen transitivamente de la clave principal.
    Existen casos prácticos en los que una desnormalización controlda puede suponer una mejora en las prestaciones de los accesos. Por ejemplo, si para realizar una determinada operación deben realizarse muchos accesos, puede resultar ventajoso una redundancia controlada de datos.

PHP y SGBD

    PHP puede conectarse prácticamente con todos los principales sistemas de gestión de bases de datos y en diferentes entornos operativos. Entre los sistemas de bases de datos más empleados están: SQLite, MySQL, mSQL, Microsoft SQL Server, Oracle y PostgreSQL.
