Formularios HTML

    El uso de formularios implica resolver los aspectos siguientes: diseño del formulario, validación primaria en el cliente y tratamiento de los datos en el servidor. Los formularios se definen con la etiqueta <form> ... </form> que tiene dos atributos obligatorios:
  • action.- indica qué aplicación procesará los datos del formulario en el servidor.
  • method.- indica el modo de transmisión de los datos. El modo GET utiliza la dirección URL para pasar los nombres y valores de los campos mientras que el modo POST envía los datos como parte de la entrada estándar (no visible en la URL).
    La etiqueta form tiene como atributos opcionales:
  • enctype.- define el mecanismo de codificación.
  • name.-
  • target.- para indicar una ventana de destino diferente a la que se utilizó para el envío de los datos.
  • accept.- tipo MIME que especifica el tipo de archivos que se pueden subir.
  • accept-charset.- especifica el juego de caracteres que el servidor utiliza para gestionar los datos del formulario.
  • autocomplete.-
    Los eventos relacionados con una acción son:
  • onclick.-
  • ondblclick.-
  • oninputchange.- se activa cuando cambia el contenido de alguno de los elementos del formulario.
  • onmousedown.- al pulsar el botón del ratón.
  • onmousemove.-
  • onmouseout.- cuando el puntero del ratón sale del área de un elemento.
  • onmouseover.- cuando el puntero pasa por el área de un elemento.
  • onmouseup.- cuando se suelta el botón del ratón.
  • onkeydown.-
  • onkeypress.- cuando se pulsa y se suelta una tecla.
  • onkeyup.- cuando se suelta una tecla.
  • onreset.- al reiniciar los elementos del formulario mediante un botón reset.
  • onsubmit.- al enviar los datos del formulario mediante el botón submit.
    A nivel formulario están los eventos:
  • onblur.- cuando un elemento pierde el foco.
  • onchange.- cuando cambia un elemento.
  • onfocus.- cuando un elemento recibe el foco.
  • onselect.- cuando se selecciona un elemento.
  • onreset.- cuando se reinician los elementos del formulario mediante el botón reset.
  • onsubmit.-
    Las etiquetas relativas a elementos del formulario son: button, details, fieldset, label, legend, input, option, optgroup, select y textarea.
  • input.- mediante el atributo type permite definir una gran variedad de elementos:

    • text.- valor predeterminado. Caja de texto.
    • password.-
    • hidden.-
    • checkbox.-
    • radio.-
    • submit.-
    • reset.-
    • image.-
    • file.-
    • telephone.- para introducir el teléfono. Hay un ejemplo en el libro con un script de validación.
    • url.- para introducir URL's. Añade automáticamente http://.
    • email.- para introducir direcciones de correo. Realiza una validación automáticamente.
    • color.- para introducir colores. Muestra una paleta de colores. Utiliza el formato RGB.
    • number.- para introducir números enteros. Admite los atributos min, max y step.
    • range.- para introducir números. Admite min, max y step. Muestra una barra deslizable.
    • datetime.- muestra controles específicos para introducir la fecha y la hora.
    • date.- entrada de fecha.
    • month.- envía año y mes.
    • week.- envía año y número de semana.
    • time.- para introducir la hora.
    • search.- similar a text pero cambia la estética.

        Además de type, la etiqueta input tiene otros atributos:

    • accept.- lista de tipos MIME relacionados con type=file.
    • alt.-
    • autocomplete.-
    • checked.- indica si el elemento está seleccionado inicialmente.
    • disabled.-
    • formaction.- dirección URL que se utiliza al activar el elemento.
    • formenctype.- modo de envío de los datos.
    • formnovalidate.- indica si el formulario no se valida antes de enviarse los datos.
    • formtarget.- ventana de destino.
    • height.- altura del control de datos.
    • list.- identificador de una lista definida con datalist.
    • max.- valor máximo que se puede introducir.
    • maxlength.- longitud máxima de la entrada.
    • min.-
    • multiple.- indica si se permiten valores múltiples.
    • name.-
    • pattern.- expresión regular que permite validar la entrada de datos.
    • placeholder.- texto guia para el usuario.
    • readonly.-
    • required.-
    • size.- cantidad de caracteres de la ventana de control.
    • src.-
    • step.- establece los valores posibles dentro de un rango.
    • type.-
    • value.- valor que devuelve el control.
    • width.-

  • label.- asigna un título al control de datos. Admite los atributos:

    • form.- para indicar el formulario propietario.
    • for.- para indicar a qué control pertenece.

  • textarea.- similar a "input type=text" pero pudiendo ocupar varias líneas. Admite los atributos:

    • autofocus.- indica si el elemento recibe el foco de forma automática al
    • cargarse la página.
    • cols.- número de columnas.
    • disabled.-
    • form.- nombre del formulario relacionado.
    • maxlength.-
    • name.-
    • placeholder.- texto guia.
    • readonly.-
    • required.-
    • rows.-
    • wrap.- comportamiento al llegar al final de la línea.

  • select.- utilizada para definir listas desplegables. Admite los atributos:

    • autofocus.- indica si el elemento recibe el foco automáticamente al
    • cargarse la página.
    • disabled.-
    • form.- nombre del formulario relacionado.
    • multiple.-
    • name.-
    • size.-

  • option.- define los elementos de la lista. Admite como atributos:

    • disabled.-
    • label.-
    • selected.-
    • value.- valor que devuelve la opción.

  • optgroup.- permite agrupar opciones de la etiqueta option para facilitar la elección de un valor. Permite, por ejemplo, deshabilitar todo un grupo de opciones. Admite los atributos:

    • disabled.-
    • label.- indica la etiqueta del grupo.

  • datalist.- permite definir una lista de opciones. Se utiliza en conjunto con "input list=" . . . ", que define el control de entrada de datos, para definir los valores de los elementos.

  • button.- permite definir botones de tipo submit, reset o button. Admite los atributos:

    • autofocus.-
    • disabled.-
    • form.-
    • formaction.- dirección URL que se utiliza al activar el elemento.
    • formenctype.- modo de envío de datos.
    • formnovalidate.- indica que el formulario no se valida antes del envío de datos.
    • formtarget.- ventana de destino.
    • name.- nombre de la variable asociada al elemento.
    • value.- valor que devuelve.

  • fieldset.- permite agrupar datos como ayuda al usuario dibujando un borde de límite alrededor de los elementos incluidos. Admite los atributos:

    • disabled.-
    • form.-
    • name.-

  • legend.- dentro del bloque fieldset, actúa como un título para el grupo de controles.

  • progress.- permite representar barras de progreso de tareas en ejecución. Incluyendo una etiqueta span se muestra un número del nivel de avance. Admite los atributos:

    • max.- valor máximo para terminar la tarea.
    • value.- indica el grado de avance de la tarea.

  • meter.- permite representar un valor escalar dentro de un rango determinado o de un valor fraccional. Por ejemplo puede servir para indicar el grado de carga de la batería. Admite los atributos:

    • max.- por defecto vale 10.
    • min.- por defecto es 0.
    • value.- valor vigente.
    • low.- valor bajo.
    • high.- valor alto.
    • optimum.- valor optimo.
