Ejemplo de estilo de impresión

    A veces se necesita volcar el contenido de la pantalla hacia una salida impresa, como puede ocurrir con el caso de recibos, facturas, etc. Otras veces la visualización se hace a través de teléfonos móviles, lo que no deja de representar nuevos desafios para los diseñadores. CSS dispone de un grupo de funcionalidades que permite personalizar el formateo de un documento para los distintos tipos de medios.

Tipos de medios

    CSS permite definir estilos orientados a un determinado tipo de medio o un grupo de tipos de medios. Cuando no se especifica nada se presupone que el estilo es válido para todos los medios (all). Los tipos de medios soportados se pueden considerar como una lista abierta dada su rápida evolución. Los más importantes del momento son:
  • aural.- relacionado con la producción de voz y sonido.
  • braille.- dispositivos de lectura táctil.
  • embossed.- impresoras braille.
  • handheld.- para dispositivos tipo PDA.
  • print.- salidas impresas.
  • projection.- proyector de transparencias.
  • screen.- pantalla de ordenador.
  • tty.- salidas del estilo teletipo.
  • tv.- televisión.
  • all.- cualquier medio.
    Para especificar el tipo de medio pueden utilizarse varias posibilidades: la regla @media, el atributo media en la etiqueta style, el atributo media en la etiqueta link y la importación de una hoja de estilo.

    La regla @media sirve para especificar que un bloque de estilo se aplica solo a un tipo de medio (o un grupo de tipos). La sintaxis tiene el formato:

          @media  tipo_medio  [, tipo_medio ...]  {definición del estilo}

    Por ejemplo, para definir un bloque de estilo para la PDA y para el proyector, sería de la forma:

          @media  handheld,  projection  {
                                           body  {font-size: 'xx-large'; color: blue;}
                                           p        {font-size: 'large'; color: red;}
          }

    Utilizando el atributo media de la etiqueta style, el formato es:

          <style  media="tipo_medio  [, tipo_medio ...]">
             . . .
          </style>

    Así, el ejemplo anterior quedaría de la forma:

          <style  media="handheld,  projection">
                body  {font-size: 'xx-large'; color: blue;}
                p        {font-size: 'large'; color: red;}
          </style>

    El atributo media de la etiqueta link se usa enlazando con un archivo externo de definición de estilos, por ejemplo:

          <link rel="stylesheet"  href="estilos.css"  type="text/css"  media="print" />

    También se puede importar una hoja de estilos externa utilizando la sintaxis:

          @import  url("archivo.css")  print;

    donde "print" sería el tipo de medio.

    La regla @import es una orden de CSS por lo que se debe incluir dentro de una sección de estilo del documento.

Configuración de un documento para impresión

    Hay que hablar del concepto de cuadro de página en CSS que contiene dos áreas principales:
  • Área de página.- donde se colocan todos los elementos de la página.
  • Área de márgenes.- zona que rodea por los cuatro lados al área de página.
    La regla @page permite establecer el tamaño de la página, los márgenes y la orientación; para complementar esta definición se utiliza la propiedad page dentro de la definición del estilo, quedando así relacionadas las dos definiciones. Esta regla se usa de la forma:

          @page  pagina1 {size: 17cm  21cm;}

    y después se referencia en la definición del estilo:

          body,  p  {page: pagina1;}

    En el ejemplo se define una página con el nombre pagina1 con un tamaño de 17 cm. de ancho y 21 cm. de alto; posteriormente las etiquetas body y p hacen referencia a esa página por su nombre. El nombre de la página es opcional y solo es necesario si hay más de una definición de páginas.

    Las dimensiones de la página se definen con la propiedad size que tiene dos valores: el ancho y la altura. Por ejemplo:

          @page  {size: 17cm  21cm;}

    También se pueden especificar los siguientes valores simbólicos:
  • auto.- valor predeterminado de la impresora vigente.
  • landscape.- gira la página e intercambia las dos medidas.
  • portrait.- reemplaza los valores predeterminados de la configuración con los valores establecidos en la propiedad size.
    Dado que no todos los navegadores soportan la regla @page es conveniente comprobar su funcionamiento antes de depender de que el código dependa de la regla.

    Para asignar el margen de la página se utiliza la propiedad margin. Los márgenes de la página no deben confundirse con el margen que se puede definir a nivel de etiqueta body. Un ejemplo sería:

          @page  {size:  17cm  21cm;
                         margin:  .8cm;}

    Las marcas de corte se utilizan para señalar la posición de corte y son signos + que se imprimen en los bordes de las páginas. La propiedad marks, que aún no está soportada por los principales agentes de usuario, tiene la sintaxis:

          marks:  crop | cross | none | inherit ;

    CSS incluye tres propiedades que permiten controlar el salto de página:
  • page-break-before.- determina como se produce el salto de página antes de un determinado elemento y en qué lado continua el flujo de la página siguiente.
  • page-break-after.- determina como se produce el salto de página después de un determinado elemento y en qué lado continua el flujo de la página siguiente.
  • page-break-inside.- determina como se produce el salto de página desde dentro de un elemento.
    Los valores que admiten las dos primeras propiedades (la tercera está soportada por pocos navegadores) son:
  • inherit.- se hereda el valor del elemento padre.
  • auto.- permite que el agente de usuario pueda insertar saltos de página donde lo considere oportuno.
  • avoid.- ordena al agente de usuario que no inserte saltos de página ni antes ni después del elemento vigente.
  • left.- fuerza uno o varios saltos de página para obtener una página izquierda en blanco.
  • right.- fuerza uno o varios saltos de página para obtener una página derecha en blanco.
  • always.- indica al agente de usuario que fuerce un salto de página o (?) después del elemento vigente.
    Por ejemplo, para forzar un salto de página antes o después de cada aparición de un elemento o de una clase determinados, se puede utilizar:

          .controlsalto  {page-break-before: always;}

    y en la página se utilizaría esta clase con el código:

          <p  class="controlsalto"> . . . </p>

    Los saltos de página no se pueden implementar dentro de los elementos posicionados. Por ejemplo, si tenemos un elemento div posicionado de modo absoluto no podremos incluir saltos de página dentro del bloque div.

    Las propiedades CSS widow y orphan permiten controlar las líneas viudas y huérfanas respectivamente. Admiten los valores:
  • inherit.- asume el valor del elemento padre.
  • valor.- número entero que indica la cantidad mínima de líneas que pueden quedar dentro de un lado del párrafo.
    Por ejemplo, con la siguiente codificación se indica que una viuda debe estar formada por al menos dos líneas; si esto no es posible, el párrafo se mantiene completo dentro de una página:

          p  {widow: 2;}

    La impresión a dos caras puede conllevar el problema de tener que definir un estilo para la página izquierda y otro para la derecha (por cambiar los márgenes, por ejemplo). Esto se resuelve con las pseudo clases :left y :right:

            @page  :left     {margin-left: 1cm;    margin-right: 0.5cm;}
            @page  :right  {margin-left: 0.5cm;  margin-right:  1cm; }
