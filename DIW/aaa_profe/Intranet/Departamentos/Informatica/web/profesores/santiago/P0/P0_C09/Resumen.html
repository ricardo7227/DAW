    Cuando se habla de contenido multimedia en las páginas web hay que diferenciar entre contenido externo y contenido inline. El contenido inline lo forman aquellos elementos que se incluyen en la página web y las imágenes y ficheros multimedia externos son aquellos que se enlazan a la página y que son bajados cuando el usuario "pincha" en el enlace.

    La multimedia inline aparece junto con el resto de los elementos de página lo que supone cierta comodidad para el usuario. La multimedia externa permite incluir más formatos de sonido, video, etc. ya que si el navegador no lo entiende llamará a alguna aplicación que le ayude.

    Luego, debemos incluir multimedia inline cuando se encuentre en un formato de fichero entendible para la mayor parte de navegadores y su carga no lleve demasiado tiempo.

    Es norma de ética entre los creadores de páginas web incluir tanto el formato como el tamaño de los archivos en el texto que sirve como enlace o al lado, para que sea el usuario quien decida si le conviene o no bajarlo.

Sonido, video y otros ficheros externos
    Para enlazar un fichero multimedia externo a una página web no hay más que crear un enlace a ese documento:

          <a href=" Fichero "> Fichero multimedia </a>

    Cuando el usuario "pincha" en el enlace, el navegador abre el fichero correspondiente si puede y, si no puede, bajará el fichero e intentará abrirlo con la aplicación adecuada. Estos programas se llaman aplicaciones auxiliares y pueden ser configurados en el navegador para que dependiendo del tipo de fichero se utilice un programa u otro.

    Si el navegador no es capaz de averiguar el formato de un fichero mostrará un mensaje preguntando al usuario si debe guardar el fichero en el disco o elegir un programa que entienda el contenido del fichero.

    Para saber el tipo de contenido de un fichero el navegador utiliza dos informaciones: por una parte la extensión y, por otra, cuando el navegador obtiene una página de un servidor web mira el tipo de fichero mime asociado al archivo o content-type. Este último es un código que utilizan los servidores web para informar del tipo de fichero que están enviando.

    Algunos ejemplos de content-type son:
  • text/plain.- texto normal.
  • text/html.- página web.
  • image/gif.- imagen en formato GIF.
  • video/mpeg.- video en formato MPEG.
  • application/msword.- documento de Microsoft Word.
    Tanto el navegador como el servidor web tienen tablas que relacionan las extensiones con el tipo de fichero (content-type) asociado; y el navegador tiene un dato adicional: la aplicación que debe usar con ese tipo de fichero.

Ficheros de sonido
    A continuación se exponen algunas características de los formatos más utilizados.

    El formato AU, creado por Sun Microsystem, es el único que es soportado por todos los navegadores. Permite varios tipos de codificación de sonido, aunque el más popular es el llamado u-law por lo que a veces al formato AU se le llama u-law. Este tipo de ficheros tiene una calidad aceptable, pero sin ser demasiado buena a causa de la codificación de 8 bits que provoca un efecto parecido al del sonido telefónico.

    Con mayor calidad están los formatos WAVE (.wav) para Windows, AIFF para Macintosh y MPEG. Los dos primeros solamente funcionan en sus respectivas plataformas y MPEG, aunque menos extendido que el formato AU, es compatible con multitud de plataformas.

    El formato Real Audio ha sido desarrollado específicamente para reproducir ficheros de sonido en Internet y en la Web. La página web de Real Audio (www.realaudio.com) ofrece el software necesario para crear y escuchar este tipo de sonido. La gran ventaja que ofrece frente al resto de los formatos es que, mientras que en éstos es necesario esperar a que llegue el fichero completo para poder escucharlo, Real Audio soporta una técnica conocida como streaming que permite escuchar el sonido según va llegando y solamente se produce una pequeña pausa al principio, antes de empezar a oirse. El inconveniente de este formato es que necesita la instalación de un servidor especial para enviar este tipo de ficheros y enlazarlos a la página requiere de una técnica especial que se describirá más adelante.

    Las extensiones asociadas a cada uno de los distintos tipos de fichero son:

EXTENSIÓN TIPO DE FICHERO
.au AU / u-law
.aif / .aiff AIFF / AIFC
.wav WAVE / WAV
mp2 MPEG audio

    Para insertar un fichero de sonido dentro de una página web se hará de la forma:

          <a  href="Quinta_sinfonia.au">La quinta sinfonía</a>

    Si se desea mayor calidad que la del formato AU, que es el más extendido, puede considerarse ofrecer varios formatos para que el usuario elija qué formato bajar.

Ficheros de vídeo
    El formato de video estándard para Internet es MPEG, que permite la codificación de audio y vídeo de gran calidad en ficheros pequeños.

    Otros formatos son AVI de Microsoft para Windows y QuickTime de Apple. Las extensiones asociadas a los diferentes tipos de ficheros son:

EXTENSIÓN TIPO DE FICHERO
.mpg / .mpeg
 MPEG
.mov
 QuickTime
.avi AVI

    Un ejemplo de uso podría ser:

          <a  href="los_indios.mpeg" alt="Escenas sobre indios">Los indios se echan al monte</a>

    Un truco usado en ocasiones, con el que se consigue un efecto interesante, consiste en convertir algún fotograma del vídeo a imagen y hacer que ésta sea el enlace al vídeo. Es conveniente no olvidar incluir el atributo alt.

Otros tipos de ficheros
    Los enlaces a archivos externos no están limitados a imágenes y vídeos. Internet está plagado de multitud de formatos entre los que destacan: archivos comprimidos ZIP (.zip o gz para Linux), ejecutables (.exe), archivos PDF (.pdf), multitud de imágenes creadas con infinidad de programas, etc.

    Cuando se pulsa un enlace a un fichero de uno de estos tipos, el sistema nos dará opción a guardarlo o a abrirlo con un programa externo, si es un tipo de fichero conocido.

Multimedia inline
    Hasta hace muy poco el único sistema disponible para incluir contenido multimedia en las páginas web era a través de archivos externos. En la actualidad están apareciendo nuevas etiquetas y atributos del lenguaje HTML que permiten incluir estos contenidos dentro de la página.

    Video inline
    Uno de los primeros mecanismos fue introducido por Microsoft Internet Explorer, incluyendo extensiones a la etiqueta img que permiten insertar vídeo con formato AVI. Estas extensiones fueron adoptadas también por el navegador Opera, pero por ningún otro (¿?), y tampoco por el estándard HTML 4.0. Los demás navegadores sencillamente ignoran este código.

    La extensión en la que se basa la inclusión de vídeo inline es el atributo dynsrc, cuyo uso es similar al ya conocido src, y cuyo valor será la dirección de un vídeo en formato AVI. Dentro de la etiqueta img deben especificarse los dos atributos, src y dynsrc, de forma que si el navegador no soporta la inclusión de vídeo dentro de la página mostrará la imagen. Un ejemplo puede ser:

          <img  dynsrc="video/Peli_03.mpeg"  src="imagenes/Foto_02.jpg" alt="[Mi video]">

    Para ver el resultado pinche aqui.

    Evidentemente pueden utilizarse los demás atributos de la etiqueta img tales como: align, vspace, hspace, width, height, etc. y, además de éstos, Microsoft incorporó como acompañamiento a dynsrc, y que permiten realizar ciertos controles sobre el video, los siguientes:
  • controls.- muestra el video AVI junto con una serie de controles que permiten pausarlo, iniciarlo, etc.

  • loop.- toma como valor el número de veces que se repetirá el video. Con el valor "-1" se repetirá indefinidamente.

  • start.- determina cuándo debe empezar a reproducirse el video. Puede tomar dos valores: "fileopen", que es el valor por defecto y que indica que el vídeo se reproducirá tan pronto como la página y el fichero con el vídeo lleguen al equipo, y "mouseover" que indica que el vídeo se reproducirá cuando el usuario sitúe el cursor sobre él.

  • loopdelay.- (solo Internet Explorer ?) que especifica con el valor en milisegundos la pausa que se producirá entre dos reproduciones sucesivas automáticas del video.
    El atributo lowsrc
    Este atributo es una extensión de Netscape a la etiqueta img que permite especificar una segunda imagen que servirá de previsualización de la imagen final especificada con src. La previsualización suele ser la misma imagen pero con peor calidad, por ejemplo en formato JPEG con alto grado de compresión y, por lo tanto, de mucho menor tamaño. De esta forma la imagen se cargará mucho antes y el usuario podrá hacerse una idea de cómo será la imagen completa antes de que ésta llegue. Un ejemplo puede ser:

          <img  src="foto_alta_calidad.gif"  lowsrc="baja_calidad.jpeg">

    lowsrc no ha sido aceptado por HTML 4.0 pero no causa ningún perjuicio a los navegadores que no la asumen.

    En algunos casos y con un poco de imaginación, lowsrc puede utilizarse para crear una pequeña animación basada en dos imágenes (ver ejemplo en pag. 347).

    Sonido inline
    Microsoft Internet Explorer ha introducido la etiqueta bgsound que permite incluir sonido de fondo en las páginas web. El sonido será cargado junto con la página y se empezará a reproducir sin intervención del usuario. Un ejemplo puede ser:

          <bgsound src="sonido/The_wall.mp3">

    La etiqueta produce un efecto visual en la página, y se reproduce el sonido como fondo. Para ver el resultado pinche aquí.

    Además del atributo src, que permite especificar el fichero a reproducir, están:
  • loop.- especifica el número de reproducciones. El valor "-1" especifica de forma indefinida.

  • balance.- (solo Internet Explorer) controla el balanceo entre los dos altavoces. El valor que se puede asignar está comprendido entre -10000 y 10000. Con -10000 el sonido solamente se oirá por un altavoz y con 10000 por el otro. Los valores intermedios balancearán el volumen entre ambos.

  • title.- (solo Internet Explorer) solamente a nivel informativo.

  • volume.- (solo Internet Explorer) admite un valor entre -10000 y 0. Con el valor "0" el sonido se reproducirá al 100% de la configuración del usuario.
    Es importante tener en cuenta que el sonido puede molestar al usuario y hacer que abandone la página y, por otra parte, que el tamaño (por lo general grande) de los ficheros de sonido puede incrementar desmesuradamente el tiempo de carga.

    Internet Explorer soporta tres tipos de ficheros de sonido inline: AU, WAV y MIDI.

    La etiqueta bgsound no es soportada por otro tipo de navegadores (?), ni ha sido incluida en HTML 4.0. Existen otros métodos, que se verán más adelante, para incluir sonidos en las páginas web.

Marquesinas animadas
    Una marquesina animada es una línea de texto que se desplaza por la pantalla de derecha a izquierda, de izquirda a derecha o en ambas direcciones alternativamente. La etiqueta específica de Internet Explorer que permite definir marquesinas es marquee. Esta etiqueta no ha sido aceptada por los demás navegadores (?) ni forma parte de HTML 4.0.

    La estructura de esta etiqueta es muy sencilla: consta de la etiqueta de principio, el texto que se mueve y la etiqueta de fin. Un ejemplo puede ser:
          <marquee> Me estoy movindo </marque>

    Si se muestra esta página con un navegador podrá observarse como el texto aparece por la derecha y se mueve hacia la izquierda hasta desaparecer. Y así repetidamente.

    El texto de una marquesina se mostrará siempre en una nueva línea y con las mismas características que se mostraría sin ser marquesina, es decir, conservará los atributos aplicados. Veamos un ejemplo con un encabezado:

          Esto se muestra en una línea y esto, por ser marquesina, en otra

    La etiqueta marquee consta de una serie de atributos que permiten cambiar la apariencia de la marquesina:
  • bgcolor.- permite cambiar el color del fondo.

  • height y width.- determinan el alto y el ancho de la marquesina. El valor puede darse en pixels o en porcentajes y no afecta al tamaño del texto.

  • hspace y vspace.- determina los márgenes entre la marquesina y su contenido.

  • align.- especifica el alineamiento de la marquesina con respecto al texto que la rodea, de forma similar a como ocurre con las imágenes. No modifica la posición del texto dentro de la marquesina, que siempre se sitúa en la parte superior. Los posibles valores son: top, middle, bottom, left y right.
    Un ejemplo de uso de estos atributos puede ser:

          <h1>
             <marquee align="right" bgcolor="#0000FF" height="20%" width="60%">
                ESTAMOS TAN AGUSTITO
             </marquee>
          </h1>

    cuyo resultado es el siguiente:

ESTAMOS TAN AGUSTITO

    Puede probarse a visualizar el ejemplo con Internet Explorer para comprobar si hay diferencias.

    El comportamiento, dirección del movimiento, número de repeticiones, etc. del contenido de la marquesina puede modificarse mediante los atributos:
  • behavior.- permite cambiar el comportamiento del contenido de la marquesina. Sus posibles valores son:

    • scroll: es el valor por defecto. Su comportamiento lo conocemos:   SCROLL

    • slide: el contenido deja de moverse cuando llega al borde izquierdo:   SLIDE

    • alternate: el contenido rebotará en los extremos:   ALTERNATE

  • direction.- indica la dirección y sus posibles valores son left (por defecto) y right.   DIRECTION="RIGHT"

  • loop.- permite especificar el número de repeticiones. Los valores "-1" e "infinity" indican indefinidamente.   LOOP="4"

  • scrollamount.- aunque haya sensación de movimiento, el contenido se desplaza a pequeños saltos. Con este atributo se especifica el número de pixels que habrá entre salto y salto. Junto con el siguiente permite controlar la sensación de movimiento y la velocidad.
    UNO   CINCO   DIEZ   QUINCE   VEINTE

  • scrolldelay.- determina el tiempo de espera entre salto y salto, en milisegundos.
    SCROLLAMOUNT="2" SCROLLDELAY="200"     SCROLLAMOUNT="2" SCROLLDELAY="50"

    Además, Netscape introdujo la etiqueta <blink> ... </blink> que muestra el texto parpadeando. Este efecto puede ser molesto o causar distracción.

Controles multimedia
    Existen controles multimedia que permiten insertar gráficos y efectos multimedia en las páginas web. Destacan, aunque aquí no se traten por salirse de los objetivos de este resumen:
  • behaviors.- comportamientos especiales de los controles y otros elementos de la página.
  • effects.- aplica un filtro gráfico a cualquier elemento de una página.
  • hot spot.- permite especificar acciones a realizar al pulsar en diferentes zonas de la ventana del navegador.
  • mixer.- mezcla varios ficheros de sonido WAV.
  • path.- permite crear movimiento de objetos por un camino establecido.
  • sequencer.- facilita el control del tiempo en los eventos de la página.
  • sprite.- sirve para crear animaciones.
  • sprite buttons.- sirve para crear botones animados.
  • structured graphics.- permite incluir gráficos que pueden ser cambiados de tamaño y rotados por el usuario.
Plug-ins y objetos incrustados
    Los plug-ins pueden considerarse como programas auxiliares que trabajan junto con el navegador ofreciéndole nuevas características. Por ejemplo, un plug-in de Excell permitiría mostrar hojas de cálculo de esta aplicación directamente en la página web, un plug-in de vídeo podría permitir abrir ficheros de vídeo de formatos distintos a los vistos, etc.

    Los plug-ins se ejecutan junto con el navegador añadiéndole capacidades inline.

    Actualmente existen numerosos plug-ins de audio y vídeo disponibles que añaden soporte para AU, AIFF, WAV, MIDI y QuickTime. Hay que tener en cuenta que si se utilizan las capacidades de algún plug-in en la página, los visitantes deberán acceder con un nevegador que admita plug-ins y que tenga el plug-in correspondiente.

    Los plug-ins más conocidos y usados son:
  • Acrobat Reader.- para mostrar ficheros de tipo pdf.
  • Shockwave.- permite reproducir vídeos generados con Macromedia Director.
  • RealAudio y RealVídeo.- para reproducir audio y vídeo en estos formatos especialmente diseñados para la WWW.
    Junto con éstos, es posible encontrar numerosos plug-ins dedicados a mostrar contenido multimedia y mundos virtuales VRML.

    Cómo usar los plug-ins
    Es primer paso para usar un plug-in es crear el objeto que se quiere insertar en la página. El proceso de creación depende del tipo de objeto. Por ejemplo, para archivos de sonido y vídeo son necesarios programas de edición y retoque de éstos.

    El siguiente paso es insertar en la página, usando código HTML, el objeto que ha de ser usado por el plug-in. El propio navegador al darse cuenta que no es capaz de mostrar el objeto buscará entre la lista de plug-ins que tiene instalados y llamará al que sea capaz de tratar al objeto; si no lo encuentra mostrará un mensaje o un icono indicativo que puede resultar molesto para el usuario.

    El término elegido para nombrar a estos objetos es embebded objects u objetos incrustados (o embebidos). La etiqueta con la que se incrusta objetos es embed que con el atributo src permite especificar el nombre o dirección del objeto. Otros atributos muy utilizados son width y height para especificar el ancho y alto del objeto en cuestión. Además, la etiqueta embed tiene una serie de atributos opcionales que varían enormemente según el plug-in.

    Alternativas para navegadores que no soportan plug-ins
    Si el navegador no dispone del plug-in adecuado para mostrar un objeto visualizará un error o un cuadro de diálogo que permita obtener el plug-in necesario (el atributo pluginspage=" url " permite especificar la dirección).

    Una alternativa para los navegadores que no soportan plug-ins es utilizar la etiqueta <noembed> ... </noembed> cuyo propósito puede ser:
  • Mostrar una imagen que represente el significado del objeto.
  • Incluir un enlace al fichero del objeto, es decir, incluirlo como fichero externo; de esta forma el usuario podrá verlo con el programa que desee.
  • Poner cualquier código HTML, applet de Java o elemento que ayude con la presentación de la página.
    Un ejemplo podría ser:

          <embed src="huellas.cdr"  width="640"  height="30"  alt="* * * * *">
            <noembed>
              <img  src="huellas.gif"  width="640"  height="30">
            </noembed>

    Los navegadores interpretarán un código u otro dependiendo de si entienden o no la etiqueta embed.

    Sonido inline con la etiqueta embed
    En el libro se trata este apartado desde el punto de vista de que Internet Explorer reconozca bgsound y Netscape embed pero probablemente, a estas alturas, ambos reconozcan las dos etiquetas.

    La etiqueta embed admite los atributos width (ancho), eight (alto) y autostart (inicio de la reproducción al cargar la página). Si ponemos los valores de width y eight a cero es conveniente poner autostart a valor "true" porque no se mostrarán los controles de reproducción y parada:

             <embed src="cancion.mp3"  width="0"  eight="0"  autostart="true">
               <noembed>
                 <bgsound src="cancion.mp3">
               </noembed>
