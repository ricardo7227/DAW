Instalación de MySQL

    La instalación puede hacerse desde Synaptic. Si se desea obtener una versión específica puede descargarse de www.mysql.com. MySQL puede no quedar habilitado de forma predeterminada, por lo que puede ser necesario editar el archivo de configuración de PHP (php.ini). PHP también debe acceder a la biblioteca cliente de MySQL "libmysql.dll" que se incluye en la distribución de PHP para que se pueda establecer la comunicación PHP-MySQL; esta dll debe estar disponible en la ruta de acceso (PATH) del entorno Windows. Muchas veces se coloca la biblioteca en la carpeta del sistema Windows (System32) pero no es recomendable.

    Por defecto, en la instalación de MySQL, se crea un usuario administrador "root" sin clave que permite empezar a administrar. Entre las primeras operaciones estará la asignación de una contraseña a este usuario.

Comprobaciones previas

    Para comprobar si está bien instalado PHP y MySQL se puede abrir una página PHP que contenga un acceso a MySQL, por ejemplo:

<?php
  echo "<b>Conexión inicial con el servidor MySQL</b><br>";
  $con = mysqli_connect("localhost", "root", "cmadrid") or
    die("No se pudo establecer la conexión con el servidor MySQL");
  echo "¡BIEN!, Conexión establecida.\n";
?>

    Si no se establece la conexión lo primero que hay que verificar es si se ha iniciado el servidor MySQL.

    No se obtiene la página php

    En el funcionamiento del ejemplo anterior intervienen varios elementos:
  • El servidor de páginas web. Puede que funcione bien con páginas html pero no con páginas php.
  • El intérprete PHP. Si la conexión servidor web-intérprete php está bien configurada debería obtenerse una respuesta al realizar la prueba con phpinfo().
  • El servidor MySQL. Se encarga de ejecutar las peticiones que le llegan desde el cliente (PHP). Quizá haya que revisar la configuración que relaciona PHP con MySQL. MySQL puede funcionar desde la línea de comandos o desde otro cliente y no desde php.
    Los elementos a revisar residen generalmente en los siguientes archivos:
  • http.conf.- configuración del servidor Apache.
  • php.ini.- configuración del interprete PHP.
  • my.ini o my.cnf.- configuración de MySQL.
  • Biblioteca elegida.- llibmysqli.dll.
  • Funciones utilizadas de la API de PHP en el script.

Operaciones básicas con MySQL

    Establecer la conexión con una base de datos

    MySQL funciona con una estructura cliente / servidor. El puerto predeterminado que utiliza MySQL para atender las peticiones es el 3306.

    En este capítulo y en el siguiente se van a utilizar las funciones de las extensiones mysql y mysqli; esta última es una versión mejorada, con mayores funcionalidades.

    La función que permite establecer la conexión con una base de datos es mysqli_connect(), cuya sintaxis es:

bool = mysqli_connect([servidor[, usuario[, password]]]);

    La respuesta es un valor booleano que depende de si se establece la conexión. Los tres párametros son opcionales, si no se informa del servidor se presupone "localhost", si no se informa del usuario se tomará el propietario del proceso y si no se informa la password se presupondrá vacía. Como servidor puede especificarse además un puerto con la forma 'servidor:puerto' o una ruta de acceso, '/ruta1/ruta2' del localhost.

    Creación de una base de datos

    MySQL suministra la función mysqli_query() que permite ejecutar instrucciones de SQL de cualquier tipo. Su sintaxis es:

int = mysqli_query($conexion, $sentenciaSQL);

    $conexion es la conexión abierta con mysqli_connect().
    $sentenciaSQL contiene la sentencia SQL en lenguaje SQL puro.

    Un ejemplo:

<?sql
  echo "<b>Creación de una base de datos</b><br>";
  $con = mysqli_connect("localhost", "root", "cmadrid")
         or die("No se pudo establecer la conexión");
  $sql = 'CREATE DATABASE Agenda';
  if (mysqli_query($con, $sql)) {
     echo "Se ha creado la base de datos Agenda\n";
  } else {
     echo "Se ha producido un error al crear la Agenda" .
          mysqli_error($con) . "\n";
  }
?>

    Selección de una base de datos

    La función mysqli_select_db() permite seleccionar una base de datos para empezar a utilizarla. Su sintaxis es:

bool = mysqli_select_db($conexion, 'base_datos');

    La función devuelve un valor booleano dependiendo del resultado de la operación. Pueden producirse errores de diferente índole: que no exista la base de datos, que se encuentre bloqueada, etc. Un ejemplo sería:

<?sql
  echo "<b>Selección de una base de datos</b><br>";
  $con = mysqli_connect("localhost", "root", "cmadrid")
         or die("No se pudo establecer la conexión");
  $db = mysqli_select_db($con, 'Agenda');
  if (!$db) {
     die ('No se puede seleccionar la Agenda:' . mysqli_error($con));
  }
  echo "¡ Operación de selección perfecta !";
?>

    Creación de una tabla

    Para crear una tabla en MySQL desde PHP se utiliza una cadena SQL con la definición de la tabla y la función mysqli_query() para enviar la sentencia a la base de datos activa o a otra base de datos utilizando el indicador de enlace correspondiente. La sintaxis de la instrucción es:

$var = mysqli_query ($conexion, $instruccion[, $resultado]);

    $instruccion es una variable cuyo contenido es la instrucción SQL de creacción de la tabla.
    $resultado es la constante MYSQLI_USE_RESULT o MYSQLI_STORE_RESULT.

    Sintaxis de la instrucción CREATE TABLE de MySQL

    La sintaxis simplificada con las claúsulas más habituales es:

CREATE TABLE nombreTabla (
             nombreCampo tipoDato [[NOT ]NULL] [DEFAULT valor]
             [AUTOINCREMENT]
             [, PRIMARY KEY([nombre_campo])]
             [, INDEX [nombre_indice] (nombre_campo)]
             [, UNIQUE [INDEX] [nombre_indice] (nombre_campo)]
             [, FOREIGN KEY nombre_campo (nombre_indice)]
             [ . . . los demás campos . . .])

  • tipoDato.- uno de los posibles tipos de datos vistos en el capítulo 3.
  • AUTOINCREMENT.- solo puede haber un campo así dentro de la tabla.
  • DEFAULT.- permite asignar un valor por defecto para cuando no se informa el campo.
  • NOT NULL.- no permite que quede sin informar.
  • NULL.- campo con valor NULL.
  • PRIMARY KEY.- no puede haber valores repetidos en este campo.
    Un ejemplo:

<?sql
  echo "<b>Creación de una tabla</b><br>";
  $con = mysqli_connect("localhost", "root", "cmadrid")
         or die("No se pudo establecer la conexión");
  $db = mysqli_select_db($con, 'Agenda');
  if (!$db) {
     die ('No se puede seleccionar la Agenda:' . mysqli_error($con));
  }
    // Se elimina la tabla si existe
  $sql = "DROP TABLE Personas;";
  $tb = mysqli_query($con, $sql);
  $sql = "CREATE TABLE 'Personas' (
                       'dni' VARCHAR(8) NOT NULL,
                       'nom' VARCHAR(40) NOT NULL,
                       'eda' SMALLINT(3) NOT NULL,
                       PRIMARY KEY('dni'));";
  $tb = mysqli_query($con, $sql);
  if (!$db) {
     die ('No se pudo crear la tabla Personas:' . mysqli_error($con));
  }
  echo '¡ Todo perfecto !';
?>

    Inserción de filas en una tabla

    La inserción puede hacerse con mysqli_query() haciendo uso de la instrucción SQL INSERT cuya sintaxis simplificada, con las claúsulas más habituales, sería:

INSERT INTO nombre_tabla [(campo1, ...)] VALUES (valor1, ...);

    Un ejemplo sería:

<?php
  echo "<B>Insertar datos</B><BR>";
  $con = mysqli_connect("localhost","root","cmadrid") or
         die ("No se puede establecer la conexión");
  $db = mysqli_select_db($con, 'Agenda');
  $sql = "INSERT INTO Personas (dni, nom, eda) Values ('";
  $sql .= mysql_escape_string("12345678") . "', '";
  $sql .= mysql_escape_string('Juan Pérez López') . "', '";
  $sql .= mysql_escape_string('45') . "');";
  $result = mysqli_query($con, $sql) or 
    die ("Error de aplicación: Inserción errónea");        
?>

    Consultas a una base de datos

    La consulta o lectura de información de una tabla o grupo de tablas de una base de datos se efectúa con la instrucción SELECT. mysqly_query() devolverá verdaderoo falso según se haya podido realizar la consulta o no. Si la consulta se ejecutó correctamente pero no se encontró ningún registro que cumpliera con el criterio de búsqueda, la respuesta será TRUE, es decir FALSE (cero) no significa que el conjunto de resultados esté vacío o no, significa que hubo un error en el acceso.

    Algunas de las razones que pueden provocar un error de acceso son:
  • Error en la sintaxis de la sentencia SELECT.
  • Problema de permisos para acceder a la tabla referenciada.
  • La tabla no existe.
  • El servidor está saturado y no puede responder.
    Un ejemplo:

$sql = "SELECT * FROM Personas WHERE dni = " . $dni1 .";";
$resul = mysqli_query ($con, $sql) or
         die("Error en el acceso a la base de datos");

    La sintaxis simplificada con las claúsulas más habituales de la sentencia SELECT es:

SELECT {lista_campos | *} FROM tabla
       WHERE condicion1
       [GROUP BY (campo1, ..) [HAVING condicion2]]
       [ORDER BY (campo2, ..) [ASC | DESC]];

    La sentencia SELECT efectua la consulta e indica si hubo o no errores, pero no devuelve directamente los datos de la consulta. Cuando devuelve no cero (TRUE) en realidad devuelve un identificador del conjunto de resultados con el que debemos recurrir a otras funciones MySQL para navegar por esa información, por ejemplo, alguna de las siguientes:
  • mysqli_fetch_array(resultado[ , tipo_resultado]).- devuelve una matriz asociativa con los valores de una fila del conjunto de resultados o falso si no quedan más filas. El tipo de resultado indica como se entrega la matriz: MYSQL_ASSOC, MYSQL_NUM o MYSQL_BOTH.
  • mysqli_fetch_row(resultado).- devuelve una matriz numerada con los valores de una fila del conjunto de resultados o falso si no quedan más filas.
  • mysqli_fetch_object(resultado[ , tipo_resultado]).- devuelve un objeto a partir de una fila del conjunto de resultados o falso si no quedan más filas. El tipo de resultado puede tomar los valores: MYSQL_ASSOC, MYSQL_NUM o MYSQL_BOTH. Las propiedades del objeto son los campos de la fila.
Al ser una matriz numerada no aparecen los nombres de los campos de la fila de datos sino únicamente su posición relativa dentro de la fila.
    Siguiendo con el ejemplo anterior, después de seleccionar un conjunto de filas, se utiliza una de las funciones anteriores:

$sql = "SELECT * FROM Personas WHERE eda > 35;";
$resul = mysqli_query($con, $sql) or
         die("Error en la instrucción SELECT");
while ($fila = mysqli_fetch_array($resul, MYSQL_ASSOC)) {
     printf("D.N.I.: %s Nombre: %s Edad: %s", $fila["dni"], $fila["nom"], $fila["eda"]);
}

    Actualización de datos

    La sentencia UPDATE de SQL permite realizar las modificaciones de los campos de una tabla en base a una determinada condición WHERE. La función mysqli_affected_rows() informa de cuantas filas se vieron afectadas por la actualización. Esta última función también se puede utilizar después de inserciones y eliminaciones. Un ejemplo sería:

$sql = "UPDATE Personas SET nom = 'Santiago Alonso Gómez', eda = '48'
       WHERE dni = '11111111';";
$resul = mysqli_query($con, $sql) or
       die("Error en la modificación de la base de datos");

    La sintaxis simplificada de la sentencia UPDATE, con las claúsulas más habituales, tiene la forma:

UPDATE tabla SET campo1 = valor1, ... WHERE condicion;

    Eliminación de filas

    La sentencia DELETE de SQL permite eliminar filas de una tabla en base a una determinada condición WHERE. La función mysql_affected_rows() informa de cuantas filas se vieron afectadas por la actuación. La sintaxis simplificada de DELETE, con las claúsulas más habituales, es:

DELETE FROM tabla WHERE condicion;

    Un ejemplo sería:

$sql = "DELETE FROM Personas WHERE dni = '22222222';";
$resul = mysqli_query ($con, $sql) or
         die("Error al borrar filas de la base de datos");


    En las sentencias UPDATE y DELETE no se debe olvidar la claúsula WHERE pues de contrario la operación se realizará sobre todos los registros de la tabla.

    Filas afectadas

    Si la operación SQL realizada fue INSERT, DELETE o UPDATE y la operación ha dado como resultado TRUE puede preguntarse a MySQL cuántas fueron las filas afectadas. La función mysqli_affected_rows() muestra dicha información:

$filas_utilizadas = mysqli_affected_rows();

    Si hay varias conexiones activas deberá incluirse la conexión como parámetro de la función.

    Con la sentencia SELECT debe utilizarse mysqli_num_rows($resultado). El parámetro $resultado es obligatorio y es el objeto resultado de la operación:

$filas_seleccionadas = mysqli_num_rows($resultado);

Seguridad en servidores MySQL

    El servidor MySQL recibe las peticiones, las procesa y devuelve los resultados al cliente. En cada petición que llega al servidor se verifica que el usuario que realiza la petición tiene permiso para mantener la conexión con la base de datos.

    Si el usuario tiene permiso de acceso, en una segunda fase de control se comprueba además que el usuario tenga los privilegios adecuados para realizar la operación requerida sobre las tablas referidas en la sentencia.

    MySQL tiene un superusuario predeterminado (root) que en el momento de la instalación no posee contraseña y una de las primeras operaciones es asignarle una con alto grado de complejidad.
