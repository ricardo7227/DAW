Estándares y extensiones del lenguaje HTML
    Para coordinar o estandarizar la interpretación de los navegadores se creó el World Wide Web Consortium (W3C). El primer estándard serio, reconocido por todos los navegadores fue HTML 2.0. El siguiente estándard importante fue HTML 3.2 y actualmente está totalmente reconocido HTML 4.0 y en proceso de incorporación el 5.0. Por otra parte, han ido surgiendo otras tecnologías como CSS, JavaScripts, Applets Java, Flash, etc. cuyo soporte varía de unos exploradores a otros.

    XHTML es una variante de HTML, en concreto es idéntica a HTML 4.0.1 pero con una serie de restricciones para ser compatible con XML. Las principales normas que hay que cumplir para que una página HTML sea compatible con XHTML son:
  • La primera línea del fichero debe ser: <?xml version="1.0"?>.
  • Todas las etiquetas tienen que tener instrucción de principio y de fin. No está permitido no poner </p> o </li>.
  • Las etiquetas de una única instrucción deben terminar en "/>. Por ejemplo <img src="f.jpg"/>.
  • Todos los valores de los atributos deben ir entrecomillados.
    Van a ir adquiriendo importancia nuevas tecnologías que añaden  características como dinamismo e  interacción con el usuario. Entre estas tecnologías nos encontramos con los ya habituales lenguajes de scripts (JavaScript y VBScript) que se insertan en las páginas, junto con el código HTML, y que permiten aumentar la interactividad con el usuario respondiendo a acciones como pulsar un botón, pasar con el ratón por encima de una imagen ,etc.

    También hay que mencionar las hojas de estilo que facilitan el mantenimiento de la apariencia de las páginas web y nos ofrecen otras posibilidades.

    La suma de HTML con los lenguajes de script y las hojas de estilo ha sido denominado HTML dinámico.

    La mayor parte de las etiquetas vistas hasta ahora pertenecen al estándard HTML 2.0. El W3C ha puesto un servicio público que permite ver si una determanada página cumple los estándares, a través de la página http:validator.w3.org. También existe un programa libre llamado Tidy que permite validar en local y está disponible en http://tidy.sourceforge.net.

    Cuando queramos asegurarnos de que cierta información pueda verla todo el mundo debemos utilizar el stándard 2.0. Por lo general, cuando un navegador se encuentra con una etiqueta o atributo que no reconoce sencillamente lo ignora.

    A continuación se muestran algunas etiquetas y atributos aparecidos con el estándard HTML 3.2.

Listas no ordenadas
    El atributo type, que puede acompañar tanto a la etiqueta ul, afectando a toda la lista, como a la etiqueta li, y afectará solamente al elemento concreto, permite elegir el símbolo que se muestra delante del elemento de la lista. Los posibles valores que puede tomar son:

  • circle.- muestra una circunferencia (vacía).
  • square.- muestra un cuadrado.
  • disk.- muestra un círculo relleno. Es el utilizado por defecto.

Listas numeradas
    Aquí el atributo type identifica el tipo de numeración. Los posibles valores que puede tomar son:
  1. 1.- numeración decimal. La establecida por defecto.
  2. a.- alfabético en minúsculas.
  3. A.- alfabético en mayúsculas.
  4. i.- romano en minúsculas.
  5. I.- romano en mayúsculas.
    Al igual que en las listas no ordenadas, el atributo type puede utilizarse con la etiqueta ol o con li, como puede verse encima.

    Para estas listas también se utiliza el atributo start que permite especificar a partir de qué número o letra empezar a numerar. Los posibles valores son los números naturales (1, 2, 3, ...). En la lista anterior se ha especificado <ol start="11">. Este atributo no puede ser utilizado de forma independiente para cada elemento de la lista, esa función la realiza el atributo value. Por ejemplo, puede ponerse la lista anterior con números romanos impares:
  1. 1.- numeración decimal. La establecida por defecto.
  2. a.- alfabético en minúsculas.
  3. A.- alfabético en mayúsculas.
  4. i.- romano en minúsculas.
  5. I.- romano en mayúsculas.
    Finalmente, el atributo compact, que puede utilizarse en cualquier tipo de lista (ul, ol, menu, dir, dl) indica al explorador que intente comprimir la lista al máximo, aunque generalmente no se aprecia diferencia.

Alineación
    Ya se ha visto que algunas etiquetas como img, p, table, ... admiten el atributo align y que la etiqueta <center> ... </center> permite centrar los elementos.

    Con HTML 3.2 apareció la etiqueta div que permite hacer divisiones en las páginas web y aplicar propiedades individuales a cada una de estas divisiones por separado. Esta etiqueta admite el atributo align con los posibles valores left, center y right. Con HTML 4.0 se ha añadido el valor justify que permite alinear a ambos lados. La etiqueta meta
    La etiqueta meta sirve para describir las características de la página web. Una de sus funciones más útiles es la carga automática de páginas web sin que intervenga el usuario, lo que se consigue incluyendo  entre <head> ... </head> código de la forma:

          <meta  http-equiv="refresh"  content="5"  url="http://es.gnome.org">

    Una página con el código anterior cargará la especificada con url pasados 5 segundos de haberse cargado ella misma.

    Los buscadores tienen una serie de programas llamados arañas, webbots o bots que navegan por la red buscando páginas que contengan cierta información. Entre la información que busca el webbot está el autor de la página, una breve descripción de la misma, palabras clave que la definan, etc.Estos datos pueden introducirse en la página a través de la etiqueta meta, haciendo uso de los atributos name y content. Un ejemplo sería:

          <meta  name="author"  content="Santiago Alonso">
          <meta  name="keywords"  content="html, manual, etiqueta">
          <meta  name="description"  content="El manual de html es un resumen del libro tal realizado por Santiago Alonso">

    La etiqueta meta tiene tres atributos: http-equiv, name y content, normalmente se usa una de las dos primeras para indicar el nombre de la información que se incluye y con content se especifica dicha información. El atributo http-equiv sirve para insertar información relacionada con el protocolo http. Un ejemplo de uso de este atributo ya se ha puesto anteriormente y otro podía consistir en especificar una fecha a partir de la cual la página no debe ser cargada. El atributo name tiene un significado muy general y puede ser utilizado con muchos propósitos, desde para enviar información a los buscadores como para dar información a los editores html para que incluyan información específica.

Otras etiquetas de HTML 3.2
    El estándard dice que las páginas web deben incluir en su primera línea el descriptor de tipo de documento, que también incluirá información acerca de la versión de HTML que se está utilizando. Como ejemplos se pueden tomar la primera línea de esta página, que hace referencia a la versión 4.0.1 y la que se utiliza para HTML 5 que es:

          <!DOCTYPE html>

    Otras etiquetas de HTML 3.2 aún no vistas son:
  • applet.- para insertar applets de Java.
  • style.- usada en la cabecera para introducir el código de las hojas de estilo.
  • script.- para introducir código JavaScript o VBScript.
  • form.- permite crear formularios.
  • isindex.- método antiguo utilizado para crear formularios.
