Listas en HTML
    Una lista es una enumeración de elementos, dispuestos de forma fácil de distinguir.
    Existen los siguientes tipos de listas:
  • Numerada u ordenada.- aparecen con un número delante que se inicia a 1.
  • No ordenada.- como en la que nos encontramos.
  • De glosario o definición.
  • Menús.- los elementos son frases de una sola línea.
  • Usadas con párrafos cortos.
  • De directorio.- para términos cortos que pueden mostrarse en múltiples columnas.
    Para definir una lista se utiliza una etiqueta que identifica el tipo de lista y una etiqueta para cada elemento, encontrándonos así:

    Numerada:
       <ol>
          <li> Elemento de lista numerada </li>
       </ol>

    Ejemplo (ver código):
  1. Elemento de lista numerada 1
  2. Elemento de lista numerada 2
  3. Elemento de lista numerada 3
    No ordenada:
        <ul>
          <li> Elemento de lista no ordenada </li>
        </ul>

    Ejemplo (ver código):
  • Elemento de lista no ordenada 1
  • Elemento de lista no ordenada 2
  • Elemento de lista no ordenada 3
    De glosario o definición:
       <dl>
          <dt> Término </dt> <dd> Definición </dd>
       </dl>

    Ejempo (ver código):
Carromato
Carro de madera cubierto por un toldo.
Motocarro
Vehículo de tres ruedas con cabina y una caja o remolque posterior de carga
    Otros tipos de lista

    Los restantes tipos de lista son muy poco utilizados y los navegadores pueden mostrarlos como las listas no ordenadas. Solamente haremos referencia a su código.

    Listas de menú: <menu> <li> Elemento </li> </menu>
    Listas de directorio: <dir> <li> Elemento de directorio </li> </dir>

    Anidamiento de listas

    El anidamiento de listas permite realizar diseños como el siguiente:
  1. Tema 1.- Procesadores.
    • Intel
    • AMD
    • Otros
  2. Tema 2.- Memorias.
    • RAM
    • ROM

Comentarios en HTML

    Se utilizan para recordar detalles del código, describir lo que falta por hacer, identificar y diferenciar partes del código, usos particulares, etc.

    Tienen la forma:

       <!-- Aquí va el comentario -->
