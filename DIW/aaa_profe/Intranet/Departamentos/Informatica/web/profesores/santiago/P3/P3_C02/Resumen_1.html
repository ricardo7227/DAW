Estructura del código

    El código debe indentarse (sangrarse) adecuadamente para facilitar su legibilidad. Hay que tener muy presente que el código de cualquier programa suele estar sometido a constantes revisiones y debe estar bien estructurado y ser fácilmente legible.

Comentarios en el código

    Podemos decir que los comentarios más que convenientes son prácticamente obligatorios. Deben ser concretos, concisos, explicando el bloque de código siguiente y no debe explicar lo que resulta obvio por la simple lectura de la instrucción. No deben contener información de seguridad puesto que son visibles con el código de la página.

    Las dos posibilidades existentes son:
  • // comentario.- para comentarios de una sola línea.
  • /* comentario */.- para comentarios multilínea.
    Cuando se realiza una modificación en un programa es conveniente añadir un comentario que indique la modificación y la fecha de la misma; pueden ser datos que aporten información de mantenimiento valiosa.

Nomenclatura

    No existen reglas universales para la nomenclatura de objetos o entidades de un programa pero cada equipo de desarrollo debe seguir unas normas o criterios comunes que cada programador cumplirá estrictamente.

    La principal regla de la nomenclatura es mantener la coherencia y un esquema de formato en el nombre de cada cosa (variable, función, objeto, módulo, etc.) La regla puede ser personal pero si es coherente será fácilmente entendible por los programadores.

    Nomenclatura de variables

    Se debe intentar que el nombre de las variables tenga relación con su contenido. La separación entre palabras se indica con una mayúscula y se suele utilizar un prefijo que describe el tipo de dato, por ejemplo: txtLineaDetalle, dtFechaEntrada, intContadorLineas, bRetorno, etc. otros prefijos pueden ser: str, n, etc. No conviene utilizar prefijos que no se expliquen por si solos.

    Nomenclatura de funciones

    El criterio de nomenclatura de las funciones es similar al de las variables; el prefijo suele indicar la acción principal de la función y suele provenir del inglés. Por ejemplo:
  • get.- obtiene un valor. Como ejemplos: getNro(), obtenerNro().
  • set.- asigna un valor. Como ejemplos: setNro(), establecerNro().
  • is.- devuelve verdadero o falso. Como ejemplos: isPar(), esPar().
  • add.- añade un elemento. Por ejemplo: addElemento(), agregarElemento().
  • print.- imprime un objeto. Por ejemplo: printFormulario(), imprimirFormulario().
    A continuación del prefijo se añade la descripción de la función en una o más palabras que empiezan por mayúscula. Por ejemplo:
  • isObjetoEmpleado().- adecuado para una función que comprueba si un objeto es de tipo Empleado.
  • getColor().- adecuado para una función que obtiene el color de un objeto.
    Longitud de los nombres

    Por razones de legibilidad no son adecuados los nombres demasiado cortos, sin significado aparente, ni los demasiado largos (más de 16 o 18 caracteres), por el espacio que ocupan en la línea de edición. La excepción a esta regla son los índices utilizados en bucles o matrices (i, x) y las variables que contienen coordenadas (x, y).

    Por norma general se acepta una variable con un nombre de una o dos letras si es una variable transitoria que se utiliza y se destruye en unas pocas de líneas.

    Palabras reservadas

    Como en todos los lenguajes, los nombres de variables o de funciones tienen la restricción de que no pueden utilizarse palabras reservadas.

¿ Está activado JavaScript ?

    Para verificar si javaScript está activado en el navegador puede utilizarse el siguiente código:

          <script language=javascript>
             alert ("Funciona JavaScript");
          </script>
          <noscript>
             No está activado JavaScript
          </noscript>

Scripts de JavaScripts

    Para incluir código JavaScript dentro de un documento HTML se utiliza alguno de los dos métodos siguientes:

          . . .
          <script type="text/javascript"  src="url o fichero.js" ></script>
          . . .

    Desde el punto de vista funcional, utilizar un fichero externo de tipo js es lo mismo que poner el código dentro del documento html y es recomendable por motivos de organización, mantenimiento, limpieza del código y seguridad, aparte de las ventajas de rendimiento que representa. Cuando el navegador utiliza caché coloca copias de los documentos descargados desde Internet durante cierto tiempo; cuando el navegador detecta que una referencia la tiene en cache no la descarga desde el servidor sino que utiliza la copia local. Cuando se realicen modificaciones en el fichero .js debe tenerse en cuenta que el navegador tiene que actualizar el área de cache. En las opciones de configurador del explorador pueden borrarse los ficheros temporales, la caché, etc.

    O bien incluyendo el código directamente en una etiqueta script:

          . . .
          <script  type="text/javascript">
          <! - -
             / /  Código JavaScript
          - - >
          </script>
          . . .

Bloque de instrucciones

    Cada línea de código finaliza en punto y coma (;) y los bloques de instrucciones se encierran entre llaves. Por ejemplo:

          <script type="text/javascript">
             var  var1 = 5;
             document.write ("Hola JavaScript <BR><BR>");
             if  (var1  >  2)  {
                   document.write ("La variable var1 es igual a: ");
                   document.write (var1);
             }
          </script>

    A continuación de </script> puede ponerse <noscript> . . . </noscript> con código HTML que se ejecutará cuando el navegador no admita javascript.

Variables

    Reglas para la formación de nombres de variables:
  • Pueden estar compuestos por letras en mayúsculas o minúsculas, números y el guión bajo.
  • El primer carácter no puede ser un número.
  • JavaScript diferencia entre mayúsculas y minúsculas.
    Los tipos básicos de variables son cuatro: indefinido, numérico, string y booleano y en cada momento será su contenido el que identifique su tipo, pudiendo, por ejemplo, asignar a una variable booleana asignarle el valor 5 pasando así a ser numérica.

    Para saber el tipo de una variable puede utilizarse el operador typeoff. Por ejemplo:

          var  var1 = 5;
          document.write (typeof (var1));

  • Indefinido.- es el tipo de una variable declarada sin asignarle valor; su valor también es indefinido. Es distinto de una variable no declarada, que en JavaScript provoca un error.
  • Booleano (boolean).- puede tener como valores true o false.
  • Numérico (number).- contienen algún tipo de valor numérico (entero, decimal, negativo, etc.).
  • Cadena (string).- se definen mediante la delimitación con comillas ("cadena"). Los valores 10 y "10" son de distinto tipo. Cuando en una operación aritmética interviene alguna variable de tipo string, JavaScript intentará convertirla en un número y si no puede el resultado será NaN (No an Number).
    Para facilitar la realización de ejercicios puede ser interesante conocer el funcionamiento de tres tipos de ventanas:
  • Ventana de alerta.- muestra una cadena y un botón de "Aceptar".
  • Ventana de confirmación.- muestra una cadena y dos botones: "Aceptar" y "Cancelar". Devuelve true o false según el botón que se pulse.
  • Ventana de entrada.- muestra un mensaje y permite introducir un valor que asignará a una variable.
    Un ejemplo utilizando estas ventanas sería:

          alert ("Ventana de alerta");
          alert (variable);
          conf = confirm ("Ventana de confirmación");
          entrada = prompt ("Mensaje", "Valor inicial");

Sentencias condicionales

    Sentencia if

          if  ( condición )  instrucción;

          if  (condición ) {
               instrucción 1;
                  . . .
               instrucción n;
          }

    Por ejemplo:   if  ( var1 > 3 )  var1 = var1 - 1;

    La condición que se evalúa se encierra entre paréntesis y siempre devuelve un valor booleano: true o false.

    Operadores condicionales
  • ==    Igualdad (no confundir con = que es de asignación).
  • !=     Desigualdad.
  • >      Mayor que.
  • <      Menor que.
  • >=    Mayor o igual que.
  • <=    Menor o igual que.
    El resultado de una comparación condicional siempre es true o false.

    Combinación de condiciones

    Para formar condiciones compuestas se utilizan los siguientes operadores lógicos:
  • &&   Corresponde a AND (Y).
  • ||      Corresponde a OR (O).
    Un ejemplo sería:

          if  ( ( var1 >= 5)  &&  (var2 = 17 ) ) { . . . }

    Sentencia if  else

          if  ( condición )  instrucción 1;
           else instrucción 2;

          if  ( condición ) {
                instrucción 1;
                      . . .
                instrucción n;
          } else {
                instrucción 1;
                      . . .
                instrucción n;
          }

    Sentencia switch

    La instrucción switch permite comparar el contenido de una variable con diferentes valores y tiene la forma:

          switch  ( variable ) {
             case ( valor 1): instrucción 1;
                              . . .
                              instrucción n;
                              break;
             case (valor 2): . . .
                              break;
             default: . . .
          }

    Operador ternario

    Puede utilizarse un método muy compacto como sustituto del if con la forma:

          condición  ?  instrucción si verdadero  :  instrucción si falso;

    Por ejemplo:

            var1 == 5 ? document.write ("IGUAL A 5") : document.write ("DISTINTO DE 5");

Control de bucles de ejecución

    Sentencia for

          for  ( condición inicial; condición de control; instrucción de modificación ) {
                   instrucciones;
          {

    Por ejemplo:

          for ( var  i = 0;  i < 10; i++){
                document.write (i);
          }

    Incremento / decremento de unidades

    La operación de sumar o restar 1 a una variable se presenta repetidamente en todos los programas. La forma estándar de escribirlo es la siguiente:

          var1 = var1 + 1;
          var2 = var2 - 1;

    El modo abreviado para hacer lo mismo es:

          var1++;
          var2--;

    Por extensión, también se permite la siguiente sintaxis:

          var3 = var3 + 5;
          var3+=5

    Sentencia while

          while  ( condición ) {
                      . . .
          }

    Por ejemplo:

          var saldo = 1000;
          var gasto = 5;
          while  ( saldo > 0 ) {
               saldo = saldo - gasto;
               gasto+=5;
               document.write (saldo);
               document.write (gasto);
          }

    Sentencia do  ... while

    Hace la función del until. La diferencia con la instrucción while anterior es que en este caso se ejecuta en primer lugar las instrucciones y a continuación se comprueba si la condición se cumple, con lo cual, las instrucciones del bucle se ejecutan al menos una vez. Su formato es:

          do  {
                  . . .
          }  while  (condición)

    Por ejemplo:

          var  saldo = 1000;
          var  gasto = 5;
          do  {
                 saldo = saldo - gasto;
                 gasto+= 5;
                 document.write (saldo);
                 document.write (gasto);
          } while  (saldo > 0)

Funciones

    El formato de la definición de una función es el siguiente:

          function  nombreFuncion  ( [param1] [, param2] ...) {
                           . . .
                         [ return  valorRetorno ];
          }

    Una función se puede asignar a una variable, por lo que se puede considerar otro tipo de dato.

    Parámetros de la función

    Los parámetros de una función pueden ser cadenas, números, valores booleanos y objetos JavaScript o personalizados y pueden ser pasados como variables o como literales. Una función puede tener definida una lista de n parámetros y la llamada a la misma contener un número diferente de parámetros. Si en la llamada hay menos parámetros de los que dispone la función, los últimos serán asignados con null.

    Llamada de función

    Desde el cuerpo de una función se puede llamar a otra, e incluso a la misma (recursión). Un ejemplo:

          function  miPopup (N)  {
                    alert (N);
                    return  "OK";
          }

    La recursividad se produce cuando una función se llama a sí misma. Un caso típico es la función factorial de n cuyo resultado es n * factorial de (n -1) cuya salida se produce cuando n llega a 1:

Ámbito de variables

    Una variable declarada dentro de una función es local a esa función y solo existe en el contexto de la misma. Una variable global, declarada fuera de las funciones, tiene un ámbito de todo el programa, incluso dentro de las funciones. Pueden verse los ejemplos de la página 403 y 404.

    Los parámetros de una función son tratados como variables locales de la función.

    Finalmente decir que una variable puede representar a un objeto.

Evaluación de una cadena de código JavaScript

    La función eval() permite evaluar una expresión de tipo cadena. Por ejemplo:

          iResultado = eval (cadena);

    Si "cadena" tiene una fórmula "eval" hará los cálculos y dejará el resultado en iResultado.

    Esta función permite crear una calculadora muy fácilmente. Ver pag. 404.

JavaScript y los motores de búsqueda

    En la sexta parte del libro, SEO, Optimización para los motores de búsqueda, se tratan diferentes técnicas para incentivar las visitas a un sitio web.

    Los robots que analizan y estudian el contenido de las páginas no tienen en cuenta (de momento) el código JavaScript. Si una página se basa excesivamente en código JavaScript y con escaso aporte de lenguaje HTML será casi desconocida para los buscadores.

    Los robots aprovechan la estructura del menú de navegación de cada sitio para recorrer los enlaces y descubrir las páginas componentes del sitio para seguir obteniendo información. Si el menú de navegación está codificado en JavaScript, algo habitual, el robot no lo tendrá en cuenta salvo que además se incluya información complementaria para que el robot pueda obtener esa información por otros medios. Por esto se recomienda que además del menú en JavaScript todas las páginas de nuestro sitio estén referenciadas por el modo HTML (con la etiqueta <a href=" ... ">) dentro de una página que actue como mapa general del sitio. De poco sirve una página perfectamente diseñada si es invisible para los buscadores.
