Java en las páginas web

    Con Java se podrá insertar contenido multimedia con mayor libertad y se consigue una interactuación completa con el usuario. Java es un lenguaje de programación desarrollado por Sun Microsystems. En la página de Sun (www.sun.com) pueden encontrarse tutoriales y otros materiales gratuitos.

    Una de las ventajas de este lenguaje es la posibilidad de hacer pequeños programas y ejecutarlos desde dentro de las páginas web. Estos programas se llaman applets. No es necesario conocer este lenguaje para aprovechar sus características en las páginas web. Existen gran cantidad de applets gratuitos que se distribuyen a través de Internet y que pueden ser utilizados libremente en las páginas. La propia compañia Sun M. ha creado una página (http://Java.sun.com/applets/index.html) en la que se muestran ejemplos y se distribuyen gratuitamente gran cantidad de applets. Otra forma de buscar applets es introducir en un buscador el texto "Java applets".

Inserción de applets Java

    Para insertar un applet de Java se utilizan las etiquetas <applet> ... </applet> y su funcionamiento es similar a la inserción de imágenes. El applet aparecerá en el lugar exacto que se inserte. Los tres atributos OBLIGATORIOS de esta etiqueta son: width, eight y code que se utiliza para especificar el archivo que contiene el applet. Por lo tanto, un ejemplo de inserción de un applet en cualquier lugar de la página tendrá la forma:

          <applet  code="Nombre_applet.class"  width="ancho"  eight="alto"></applet>

    Si no se especifica alguno de los tres atributos obligatorios el applet no se ejecutará.

    Y un ejemplo completo de uso de un applet podría ser:

         <html>
          <head>
             <title>Applet sonoro</title>
          </head>
          <body>
             <applet code="SoundEjemplo.class" width="450" height="50"</applet>
          </body>
         </html>

    Como puede observarse los applets Java tienen extensión .class.

    Cuando el navegador encuentra el código anterior procede a bajar el fichero especificado (si éste no se encuentra en el propio ordenador, que no es el caso) y reserva un rectángulo de las dimensiones especificadas donde se mostrará el applet. A continuación llama a una máquina virtual Java (JVM) que será la que ejecute el applet. Así, cuando el cursor se sitúe dentro del rectángulo,, el control pasará al applet que será el encargado de realizar las operaciones oportunas.

    Especificación de la dirección del applet

    El atributo code permite especificar el nombre del fichero pero no admite nombres de directorios ni direcciones URL; esta función la realiza el atributo codebase, de forma que si el applet se encuentra en el directorio "applets" el código a utilizar será:

          <applet codebase="applets/" code="SoundEjemplo.class" width="450" eight="50"</applet>

    Y si el fichero está en Internet tendrá la forma:

          <applet code="Clock.class" width="155" eight="150"
             codebase="http://java.sun.com/applets/jdk/1.4/demo/applets/Clock/"
          </applet>

   En la mayor parte de las ocasiones, los autores de los applets permiten su uso desde el servidor donde se encuentran pero no permiten su copia, aunque ésto no suponga ningún inconveniente.

    Otros atributos de la etiqueta applet

    Además de los cuatro atributos vistos anteriormente, existen:

  • name.- permite especificar el nombre del elemento y posibilita hacer referencia a él desde otros lugares de la página.

  • align.- controla el alineamiento con relación al texto circundante. Los posibles valores son: left, center y right. (no me ha funcionado).

  • vspace y hspace.- establece los márgenes superior e inferior y laterales entre el texto que bordea al applet y el applet.

  • alt.- permite asignar un texto alternativo para cuando el navegador tiene problemas para ejecutar el applet.

  • archive.- permite especificar un archivo auxiliar para el applet donde se pueden insertar las imágenes, sonidos, ... necesarios para la ejecución del applet. Fue pensado para tener en un archivo comprimido todos los archivos auxiliares del applet.

  • mayscript.- no tiene ningún valor asignado y su significado es que el applet puede ser accedido usando código JavaScript.

  • title.- permite especificar un texto que será mostrado en un bocadillo al pasar con el ratón por encima del applet.
    La etiqueta param

    Esta etiqueta consta de una única instrucción <param> (no existe </param>), debe incluirse dentro de la etiqueta <applet> ... </applet> y su función es definir los parámetros para el applet. Los parámetros son propiedades específicas de cada applet que permiten modificar sus características; constan de un nombre y un valor que son indicados usando los atributos name y value de la forma:

          <applet code=" Nombre_applet.class " width="ancho" height="alto">
             <param name=" Nombre_parametro " value=" Valor_param ">
          </applet>

    Cada etiqueta param define  un parámetro y puede haber tantas como sea necesario. Por lo general, los applets que se distribuyen gratuitamente tienen un gran número de parámetros para que puedan adecuarse a las necesidades de los programadores y suelen ir acompañados de alguna documentación.

    Un applet muy conocido es Animator.class que toma un conjunto de imágenes y las va mostrando de forma ordenada para crear un efecto de animación. Éste era el único método antes de que aparecieran los GIF's animados. En la página 372 hay un ejemplo de utilización de este applet.

    Alternativas a los applets Java

    Para aquellos navegadores que no soportan los applets de Java puede incluirse dentro de la etiqueta applet, además de las etiquetas param, otro código alternativo que sera ejecutado por estos navegadores  y no por los que lo soportan. Un ejemplo podría ser:

       <applet code="Animator.class" width="55" height="68" alt="Lo siento, no puede ver el applet">
          <param name="imagesource" value=".">
          <param name="endimage" value="10">
          <param name="pause" value="100">
          <param name="pauses" value="2500|100|100|100|100|100|100|100|100|100">
             <img src="T5.gif" width="55" height="68" alt="Duke te saluda">
       </applet>

    Como se ve, también puede utilizarse el atributo alt. Los navegadores sin soporte Java verán una imagen de Duke estática, sin movimiento.

    Un resumen de los atributos de la etiqueta applet y de la subetiqueta param es el siguiente:
  • applet.- etiqueta usada para insertar applets.
  • code.- indica el archivo que contiene el código.
  • width.- ancho.
  • height.- alto.
  • codebase.- especifica el directorio/URL donde se encuentra el archivo indicado en code.
  • alt.- texto alternativo.
  • name.- nombre del applet. Función informativa y para poder identificarlo.
  • align.- alineamiento.
  • vspace y hspace.- márgenes.
  • archive.- para especificar un archivo comprimido que contenga todos los archivos auxiliares que necesite el applet.
  • mayscrip.- especifica que el applet puede ser usado desde un lenguaje de script.
  • tittle.- para dar un título que será mostrado en un bocadillo al pasar con el puntero por encima.
  • param name="..." value="...".- para especificar parámetros.
  • [codigo html alternativo].- código HTML alternativo para navegadores que no soportan applets.

El nuevo estándard: la etiqueta object
    El consorcio W3C creó la etiqueta object que introdujo en el estándard HTML 3.2 para incluir varios tipos de ficheros multimedia, applets de Java, controles ActiveX, documentos HTML o prácticamente cualquier otro tipo de objeto. Por ejemplo, para incluir un vídeo AVI puede utilizarse el código:

          <object data="video/Peli.mpeg" type="video/mpeg" width="350" height="263">
             <img src="imagenes/Foto_02.jpeg" width="350" height="263" alt="[Mi casa]">
          </object>

    Cuyo resultado puede verse pinchando aquí.

    Dentro de la etiqueta object puede incluirse código alternativo para los navegadores que no reconozcan esta etiqueta.

    Puede verse más información sobre esta etiqueta en la dirección:

          http://www.w3.org/TR/WD-object-970218

    Una lista completa de los atributos que admite la etiqueta object es la siguiente:
  • id.- identificador
  • declare.- indica que el objeto va a ser declarado pero no instanciado.
  • classid.- URL con una implementación para el objeto.
  • codebase.- URL donde se encuentran las implementaciones.
  • data.- URL del objeto.
  • type.- tipo de objeto.
  • codetype.- tipo de objeto según el atributo classid
  • standby.- texto que se muestra mientras se carga el objeto.
  • align.- especifica la alineación. Los posibles valores son: texttop, middle, textmiddle, baseline, textbottom, left, center y right.
  • width.- ancho.
  • heigh.- alto.
  • border.- grosor del borde.
  • hspace y vspace.- espacio entre el texto de alrededor y el objeto.
  • usemap.- mapa de la imagen.
  • shapes.- zonas con enlaces.
  • name.- utilizado para los formularios.
  • title.- utilizado como aviso en ciertas circunstancias.
    Inserción de un applet Java

    La forma de insertar un applet Java utilizando la etiqueta object es similar a como se hacía con la etiqueta applet. Dentro de la etiqueta object también se incluirán las etiquetas param necesarias.

    La etiqueta param admite aquí dos nuevos atributos:

  • valuetype.- especifica el tipo de valor que recibirá el parámetro. Sus posibles valores son:

    • data: identifica a las cadenas de caracteres.
    • ref: el valor es una dirección URL que indica donde están almacenados los valores para dicho parámetro.
    • object: el valor es el nombre de otro objeto del mismo documento precedido del caracter #.

  • type.- especifica el tipo mime del valor asignado a este atributo con value.
    El ejemplo de la sección "Alternativas a los applets Java", utilizando la etiqueta object quedaría de la forma:

          <object codetype="application/octet-stream" classid="JavaTM:Animator.class"
                width="55" height="68" alt="Lo siento, no puede ver el applet">
             <param name="imagesource" value=".">
             <param name="endimage" value="10">
             <param name="pause" value="100">
             <param name="pauses" value="2500|100|100|100|100|100|100|100|100|100">
             <img src="T5.gif" width="55" height="68" alt="Duke te saluda">
          </object>

    Como puede observarse, la complejidad añadida con respecto a la etiqueta applet es la utilización del atributo codetype para indicar el tipo mime del applet, que es necesario para indicar al navegador el tipo de objeto y que pueda decidir, dependiendo de si soporta la tecnología, bajar o no el archivo.

ActiveX: la tecnología del futuro

    El objetivo de la tecnología ActiveX es similar a la de los plug-ins, insertar objetos de diferente tipo en una página web, aunque va más allá al añadir mayores posibilidades de interacción y comunicación con programas externos.

    ActiveX está compuesto por dos tipos de objetos: controles activos (Active controls) y documentos activos (Active documents). Estos últimos permiten insertar documentos con formato PDF, DOC, etc.

    Los controles ActiveX guardan parecidos con los objetos para plug-ins y con los applets Java aunque presenta algunas mejoras con respecto a ambos. Como ocurría con los plug-ins, los controles y documentos de ActiveX pueden ser insertados en una página web, sin embargo no requieren de un pequeño programilla para cada tipo de objeto ActiveX; esta nueva tecnología ha sido denominada como auto-contenida porque cada objeto tiene suficiente información para ejecutarse sin ayuda de ninguna aplicación.

    Podemos crear nuestros propios controles o usar los creados por otros programadores que nos los venden o distribuyen de forma gratuita. Existen herramientas que convierten esta programación en un juego de niños; destacan las creadas por Microsoft y distribuidas a través de www.mirosoft.com/ie/.

    Inserción de un control ActiveX

    Un ejemplo de inserción de un control ActiveX puede ser el siguiente:

          <object width="200" height="300"
                 classid="clsid:99B42120-6EC7-11CF-A6C7-00AA00A47DD2">
          <param name="angle" value="55">
          <param name="alignment" value="2">
          <param name="backstyle" value="3">
          <param name="caption" value="Curso WEBMASTER">
          <param name="fontname" value="Arial">
          <param name="fontsize" value="30">
          <param name="fontbold" value="1">
          <param name="backcolor" value="#006600">
          <param name="forecolor" value="#FFFFFF">
          [Para visualizar el control ActiveXTM necesita un navegador
             de las últimas versiones.]
          </object>

    El valor del atributo classid es el que identifica el tipo de control ActiveX que se inserta. Este código es complejo y está fuera de los objetivos de este manual.

    caption: contiene el texto a mostrar.
    angle: el ángulo.
    etc.

    Puede probarse a cambiar los valores de los parámetros y comprobar el resultado.

    Existe un plug-in para Netscape, Ncompass, que añade a este navegador la posibilidad de entender controles y documentos ActiveX.
