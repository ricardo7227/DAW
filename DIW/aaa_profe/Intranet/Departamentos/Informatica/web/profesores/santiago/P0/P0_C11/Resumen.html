    HTML ofrece la posibilidad de crear formularios e insertar cajas de texto, botones de opción y otros controles que permiten interactuar con el usuario.

Formularios en HTML

    Quizá las primeras páginas que hicieron uso de esta posibilidad fueran las de los buscadores, que utilizaron formularios muy sencillos, con un solo elemento conocido como caja de texto, donde se introduce la cadena a buscar.

    Los formularios pueden contener gran diversidad de elementos que permiten un amplio abanico de posibilidades y les confieren gran versatilidad; un formulario no es más que un conjunto de elementos que permiten al usuario introducir datos de forma sencilla.

    La etiqueta que permite crear formularios es <form> ... </form>, dentro de la que se pueden insertar todos los controles. Un ejemplo de un formulario básico, que contiene solamente una caja de texto, es el siguiente:

   <form>
       Nombre: <input type="text"/>
   </form>
Nombre:

    La novedad con respecto a las páginas anteriores es que en este caso el usuario puede introducir un texto y enviarlo al pulsar Intro. Para enviar el texto a algún sitio hay que utilizar el atributo action; en el ejemplo anterior el texto que se introduzca no se envía a ningún sitio.

    En los casos en los que el formulario conste de más de un control deberá incluirse un botón de envío de la información. Para formularios con un único elemento no es necesario y se sobreentiende que al pulsar Intro se quiere enviar.

    La etiqueta input permite insertar varios tipos de controles entre los que se encuentra el botón de envío, para el que habrá que especificar type="submit". El texto por defecto del botón depende del navegador pero puede especificarse con el atributo value.

   <form>
       Nombre: <input type="text"/><br/>
       <br/>
       <input type="submit" value="ENVIAR"/>
   </form>
 Nombre:


    Dentro de la misma página pueden existir varios botones con lo que se hace recomendable el uso del atributo name que permita al servidor conocer cuál se ha pulsado. Un ejemplo típico puede ser la inserción de botones de movimiento que permitan diferenciar entre aariba, abajo, izquierda y derecha. Así, una etiqueta input más completa tendría la forma:

          <input type="submit" value="arriba" name="b_arriba"/>

Controles de formularios

    Todos los controles que pueden insertarse en un formulario permiten introducir información: unas veces en forma de texto de una o varias líneas, oculto o visible, otras en forma de elección de una opción, etc.

    Los controles que pueden crearse con la etiqueta input se diferencian mediante el atributo type que puede tomar los valores: text, password, radio, checkbox, submit, image, reset, file y hidden. Además del atributo type, esta etiqueta puede tomar otros atributos que dependen del valor asignado a type, es decir, que dependen del tipo de control. Algunos de los atributos comunes son name, mencionado anteriormente y que se recomienda utilizar siempre y align.

    Cajas de texto

    type="text"
Texto:

    Atributos:
  • name.- asigna un nombre al control.
  • size.- tamaño de la caja. El valor predeterminado es "20".
  • maxlength.- número máximo de caracteres que se pueden escribir. No hay límite por defecto.
  • value.- texto que aparecerá por defecto y que se suele utilizar para especificar instrucciones o la respuesta más probable.
  • align.- admite los valores top, middle, bottom, left y right, que permite alinear el control con relación a la línea de texto en la que se encuentra.
    Cajas de texto para claves

    type="password"
Password:

    Similares a las cajas de texto pero al escribir se mostrarán asteriscos.

    Botones de elección

    type="radio"
Sexo:    Hombre   Mujer

    Este tipo de controles tiene dos estados: ON o seleccionado y OFF o no seleccionado, siendo el valor inicial de todos ellos OFF. Puede seleccionarse un botón inicialmente mediante el atributo checked.

    El ejemplo tiene el código:

            Sexo: &nbsp;&nbsp;
            <input type="radio" name="sexo" value="hombre">
            Hombre&nbsp;&nbsp;
            <input type="radio" name="sexo" value="mujer" checked="checked">
            Mujer

donde se puede comprobar que el atributo name coincide para asignar un nombre al grupo y el atributo value asigna un nombre diferente a cada botón. Así, se pueden crear varios grupos diferentes de botones dentro del mismo formulario.

    Cuando se envía la información del formulario al servidor, de los grupos de botones se envía el nombre del grupo y el nombre del botón seleccionado. En el caso del ejemplo se enviaría:

          sexo=mujer          o
          sexo=hombre

    Además de los atributos vistos, también tiene align.

    Cajas de selección

    type="checkbox"

    Similares a los botones de elección pero permiten seleccionar varias de las opciones existentes. Cada caja puede tener dos valores: seleccionado o no seleccionado. Para que inicialmente se encuentren seleccionados hay que utilizar el atributo checked.

<input type="checkbox" name="tenis"/>Tenis<br/>
<input type="checkbox" name="futbol"/>Fútbol<br/>
<input type="checkbox" name="atletismo"/>Atletismo
</form>

Fútbol

    Puede observarse como, a diferencia de los botones de elección, el atributo name de las cajas de selección son independientes.

    Con las cajas de selección también puede utilizarse el atributo value que, aunque no es imprescindible, a veces puede resultar conveniente. En el ejemplo anterior, si se seleccionan  los deportes Tenis y Atletismo, al servidor se le enviará la siguiente información:

          tenis=ON&atletismo=ON

    Si se utiliza el atributo value, al enviar la información se sustituye el valor ON por el asignado en value. Por ejemplo, si el input relativo al atletismo fuera:

       <input type="checkbox" name="atletismo" value="SI"/>Atletismo

    La información enviada por el navegador sería:

          tenis=ON&atletismo=SI

    También puede utilizarse el atributo align.

    Imágenes

    HTML permite una alternativa para insertar un botón de envío personalizado al permitir utilizar la imagen que se desee. Para insertar este tipo de control se utiliza el atributo type="image" junto con src="Fichero_con_imagen".

    Pueden utilizarse (aunque deben ser comprobados) los atributos width, height, vspace, hspace y border.

    Las imágenes de los formularios tienen una función extra ya que el navegador envía, cuando el usuario pulsa sobre una imagen, las coordenadas x e y sobre las que se ha pulsado, en relación a la esquina superior izquierda de la imagen, como si fueran parte de la información del formulario. De esta forma, el servidor puede realizar diferentes instrucciones dependiendo del lugar de la imagen donde se ha pulsado. De hecho este es un método alternativo para crear una imagen mapa; para ello basta con usar un código como el siguiente:

          <form>
             <input type="image" src="Imagenmapa.gif"/>
          </form>

    El servidor será el que analice la información enviada por el navegador y ejecute las instrucciones programadas.

    Otro atributo que admite el type="image" es name.

    Botón de inicialización de controles

    type="reset"
Texto:       

    Restablece a sus valores originales los controles del formulario.

    El atributo value permite cambiar el texto del botón. También admite el atributo name.

    Botón genérico

    type="button"

    Existe la posibilidad de crear un botón genérico, sin ninguna función predefinida. Por ejemplo:

          <input type="button" value="Pulseme">

    La acción a realizar deberá ser codificada en lenguajes de script como JavaScript o VBScript. Para insertar el código de estos lenguajes se utiliza el atributo onclick:

          <input type="button" value="Texto del botón" onclick="Código_de_script">

    Por ejemplo:

          <input type="button" value="Cerrar la ventana" onclick="window.close();">

    cuyo resultado puede verse pulsando aquí.

    Otro ejemplo, que permite utilizar un botón como un enlace sería:

          <input type="button" value="Cargar otra página"
             onClick="window.location.replace('11_01_basico.html');">

    cuyo resultado puede verse pulsando aquí.

    HTML 4.0 introduce la etiqueta button para insertar botones. Dispone del atributo type con tres valores: submit (envío), reset (reiniciar controles) y button (genérico). La innovación de esta etiqueta es que permite insertar en el botón imágenes o texto formateado con código HTML. Un ejemplo sería:

          <button type="reset" name="restablecer">Restablecer<img src="Imagen.gif" alt=""></button>

    El texto y la imagen entre la instrucción de inicio y la de fin se insertarán en el botón.

    Campos ocultos

    type="hidden"

    Los campos ocultos son un tipo de control que únicamente sirve para incluir información adicional entre los datos del formulario. Es un control invisible.

    Los atributos value y name indican los datos y el nombre. Por ejemplo:

          <input type="hidden" name="funcion" value="enviar_mail">

    Innovaciones de Microsoft

    Microsoft introduce en la etiqueta input una serie de atributos comunes a todos los tipos de controles:
  • title.- mostrará un bocadillo explicativo con el texto correspondiente.
  • tabindex y notab.- permiten cambiar el orden en que recibirán el foco los diferentes controles o elementos al pulsar la tecla TAB.
  • readonly y disabled.- permiten leer solamente y deshabilitar los controles, generalmente de texto.
Otros controles

    Cajas de texto multilínea: TEXTAREA

    Son similares a las cajas de texto, con la ventaja de que permiten escribir varias líneas. Una vez que estemos escribiendo texto dentro de este elemento, la tecla Intro llevará el cursor al principio de la siguiente línea.La etiqueta usada para insertar un control de este tipo es <textarea> ... </textarea>, que admite varios atributos:

  • name.- asigna un nombre al elemento y será enviado por el navegador.
  • rows.- número de líneas.
  • cols.- número de columnas.
  • wrap.- permite controlar el comportamiento del cursor al llegar al final de la línea. Admite los valores:
    • OFF.- el texto se desplaza hacia la izquierda y el cursor no saltará a la línea siguiente hasta que se pulse Intro.
    • SOFT.- cuando se llega al final del área reservada se produce un salto de línea. Cuando el navegador envíe el texto volverá a colocarlo en una sola línea.
    • HARD.- igual al anterior pero al mandar la información el navegador al servidor enviará también los saltos de línea, como si el usuario hubiera pulsado Intro al llegar al final de cada línea.
    El texto llano que se sitúe dentro de la etiqueta textarea será mostrado como contenido inicial del cuadro de texto.

    textarea también admite los atributos: align, disabled, readonly, tabindex, notab y title.

   

    Listas desplegables

    Permite elegir una o varias de las opciones mostradas. Se definen mediante la etiqueta <select> ... </select>.

     Las opciones se insertan mediante la etiqueta option. Ésta admite la etiqueta selected que permite establecer la opción u opciones seleccionadas por defecto.

<select name="color" size="3" multiple>
<option selected/>Rojo
<option/>Amarillo
<option selected/>Verde
<option/>Azul
</select>

    Otros atributos de la etiqueta select son:

size.- permite establecer el número de opciones de la lista que se mostrarán.
multiple.- permite que se seleccionen varias opciones de la lista.
align, disabled, title.- ya conocidas y que debe probarse su funcionamiento con select y con option.

Cómo usar los datos de un formulario

    Para hacer llegar los datos desde el navegador del cliente hasta el servidor es necesario poner en acción unas programas llamados CGI's, que se ejecutan en el servidor y que pueden estar escritos en una gran diversidad de lenguajes de programación.

    Para enviar la información al servidor la etiqueta form dispone del atributo action para especificar la URL del programa CGI al que tiene que enviar los datos. Además hay que utilizar el atributo method para indicar la forma en que se enviarán los datos. Este atributo puede tomar dos valores: get y post. Un ejemplo:

          <form method="post" action="http://www.tiernogalvan.es/cgi-bin/programa_cgi">
             . . . código con controles . . .
          </form>

    El navegador mandará al programa_cgi del equipo y directorio especificado los datos del formulario codificados con un código especial cuyotipo MIME es  application/x-www-form-urlencoded. La etiqueta form tiene un atributo, enctype, que permite cambiar el tipo de codifocación. Si no se quiere utilizar ninguna se puede escribir:

          <form enctype="text/plain" method="post" . . .> . . . </form>

    No usar codificación puede causar problemas al enviar caracteres especiales como son las letras acentuadas, pero puede ser útil en otras al no tener que decodificar el mensaje.

    Formularios sin CGI's

    Una alternativa a los CGI's son los lenguajes de script, que permiten interactuar con el navegante sin hacer uso del servidor. Existe otra posibilidad más simple que consiste en enviar los datos del formulario a una dirección de correo. Para ello hay que utilizar los atributos action, method y enctype de la siguiente manera:

    <form action="mailto:sagsag@hotmail.es" enctype="text/plain" method="post">
       <!-- . . . -->
    </form>

    Al pulsar sobre el botón de envío el navegador advertirá al usuario y si acepta mandará un correo con la información.

    Envío de ficheros usando formularios

    Para enviar ficheros desde el ordenador del usuario hay que utilizar en primer lugar el atributo enctype="multipart/form-data" y después el atributo action para especificar la URL del programa que recibirá el fichero en el servidor y realizará el tratamiento adecuado. También hay que utilizar un control  definido con la etiqueta input con type="file" que el navegador mostrará como una caja de texto junto con un botón que permitirá buscar y elegir el fichero a enviar.

    Un ejemplo puede ser:

<form enctype="multipart/form-data"
  action="http://localhost">
  Enviar el fichero:
  <input type="file" name="fichero"><br/>
  <input type="submit" value="Enviar fichero">
